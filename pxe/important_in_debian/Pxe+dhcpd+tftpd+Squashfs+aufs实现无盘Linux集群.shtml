Pxe+dhcpd+tftpd+Squashfs+aufs实现无盘Linux集群 
                                                                                    
关键字： 
diskless,ramdisk，initrd,Pxe,dhcpd,dhcpd3,isc-dhcp-server,tftpd,tftp-hpa,squashfs,aufs，无盘 
  
         随着计算机硬件的飞速发展，硬盘IO越来越多的已经高并发操作系统的瓶颈所在，且硬盘往往是最容易故障的硬件。有鉴于此，无盘系统就有了用武之地。一个应 用实例是当今最火热的社交网站Facebook。为了提高其庞大用户的访问体验，Facebook有45%的memory cache服务器，大多数无盘，低碳环保，节省费用，且服务器故障减少很多。 
         配置一个无盘的集群系统的优点显而易见：无盘系统性能好；可定制；维护方便；安全性高，被黑客攻击只需要重启服务器一切复原。 
  
         适用于无盘系统的情况： 
*） 内存提供。如mecached,redis,ttserver等key value store； 
*） 并行计算。如科学研究，实验室计算集群； 
*） 集中管理。如网吧，培训系统，教学系统； 
  
         Linux下的无盘系统有三种实现方式,区别在于根分区挂载方式的不同：root over nfs,root over ram,root over squash。Nfs形式的根分区是一个nfs server提供的目录；ram形式的根文件系统直接挂载到tmpfs上；而squashfs形式的根目录是一个tmpfs，而根文件系统是一个 squashfs镜像文件,gzip形式的squashfs提供了约60%的压缩比率，也就是说比tmpfs形式的根节省60%的内存，因此选择第三 种。 
         目前lzma压缩格式的squashfs已经在开发中，它可以提供67%左右的压缩比率，且压缩/解压缩的cpu消耗并不超过gzip很多。 
         笔者已经实现由一台diskless server分发定制化的diskless nodes给很多服务器的系统。Diskless nodes包括nginx+php-fastcgi,apache+mod_php,memcached,exim4,ttserver等,并在陆续将 haproxy,varnish/traffic server/squid等高并发系统往无盘方向迁移。以下详述基于debian linux 6.0的实现过程。 
  
         为无盘节点制作squashfs镜像 
#mkdir /data/netboot && cd /data/netboot 
  
用debootstrap来bootstrap出来一个基本debian系统。 
# debootstrap squeeze  ./squashfs http://221.238.197.8/debian 
I: Retrieving Release 
I: Retrieving Packages 
I: Validating Packages 
  
Chroot到新系统，做自己需要的目标系统个性化配置 
# chroot /data/netboot/squashfs  
   
  在系统环境里面一些可能动态增长的目录或者文件挂载成tmpfs或者nfs，以避免根分区被塞满。 
因为无盘节点的根分区是tmpfs，所以当其达到100%的时候系统你甚至无法去远程登陆它。 
  
例如exim4邮件系统，可将日志目录单独挂载。 
  
编辑/etc/fstab： 
  
tmpfs      /var/log/exim4        tmpfs   size=2G,rw,nosuid,mode=0755,uid=101,gid=103               0 0 
  
  
  
注意，这个系统环境将是diskless node启动之后的系统环境。因此将需要做的配置都提前做好，比如安装sshd,memcached，配置登陆密码，配置开机自动运行等等。 
  
配置结束之后将不需要的文件都删掉。例如 
# apt-get clean清除下载到本地的deb包 
清除/var下的日志，永远不会用到的系统软件等等。 
因为ramdisk启动过程用不到，所以在最后， 
/dev下其余设备可删除，只需要保留console null initctl即可。 
时刻记住，这个系统里的一切文件，在你的无盘系统上都占用内存！ 
之所以在最后做这个步骤，因为操作这个基本系统的时候可能要用到一些资源。例如安装openssh-server的时候要用到/dev/random生成随机数。 
  
退出chroot环境，并生成squashfs镜像。 
# exit 
# cd /data/netboot 
# mksquashfs ./squashfs/* squashfs.img 
  
一个apache+php环境可以将squashfs做到100M以内。 
  
diskless node的内核和ramdisk 
我们的diskless node内核没什么特殊要求，可以用自己编译的内核，也可以直接用发行版的自带内核。本文用发行版的自带内核。 
# cp /boot/vmlinuz-2.6.32-5-amd64 /data/tftpboot/vmlinuz 
  
Ramdisk需要自己折腾一下，因为发行版自带的initrd执行脚本并没有实现root over squashfs形式，而且也没有加载squashfs和aufs。 
  
重新生成带squashfs和aufs模块的ramdisk 
# echo “squashfs” >> /etc/initramfs-tools/modules 
# echo “aufs” >> /etc/initramfs-tools/modules 
# mkinitramfs 2.6.32-5-amd64 -o /data/tftpboot/initrd.gz.tmp 
  
将ramdisk解开并做一些hacking 
# mkdir /data/tftpboot/tmp && cd /data/tftpboot/tmp 
# gzip –cd ../initrd.gz.tmp | cpio –idm 
  
编辑 init脚本，在/proc/cmdline的cases里面多加一个启动方式。将方框里的内容加在这一段之后 
         nfsroot=*) 
                 NFSROOT="${x#nfsroot=}" 
                 ;;  
         sfsroot=*) 
                 SFSROOT="${x#sfsroot=}" 
                 ;; 
  
在scripts/下添加一个sfs文件，内容如下： 
retry_nr=0 
  
do_sfsmount() 
{ 
         configure_networking 
  
         if [ "x${SFSROOT}" = "xauto" ]; then 
                         SFSROOT=${ROOTSERVER}:${ROOTPATH} 
                 else 
                         SFSROOT=${ROOTPATH} 
                 fi 
  
         fi 
  
         [ "$quiet" != "y" ] && log_begin_msg "Running /scripts/sfs-premount" 
         run_scripts /scripts/sfs-premount 
         [ "$quiet" != "y" ] && log_end_msg 
  
         if [ ${readonly} = y ]; then 
                 roflag="-o ro" 
         else 
                 roflag="-o rw" 
         fi 
  
         [ -d /media ] || mkdir /media 
  
         [ -d /media/0 ] || mkdir /media/0 
         mount -t tmpfs squash /media/0 
         wget "${SFSROOT}" -O /media/0/squashfs.img                fi 
  
          
         modprobe -q loop 
         modprobe -q nfs 
         udevadm settle 
         losetup /dev/loop0 /media/0/squashfs.img 
         [ -d /media/1 ] || mkdir /media/1 
         mount /media/0/squashfs.img /media/1 
  
         [ -d /media/2 ] || mkdir /media/2 
         mount -t tmpfs -o size=100M tmpfs /media/2 
  
         mount -t aufs -o br:/media/2:/media/1=ro aufs ${rootmnt} 
  
         [ -d ${rootmnt}/media/0 ] || mkdir -p /root/media/0 
         [ -d ${rootmnt}/media/1 ] || mkdir -p /root/media/1 
         [ -d ${rootmnt}/media/2 ] || mkdir -p /root/media/2 
  
         mount -n -o move /media/0 ${rootmnt}/media/0 
         mount -n -o move /media/1 ${rootmnt}/media/1 
         mount -n -o move /media/2 ${rootmnt}/media/2 
} 
  
mountroot() 
{ 
         [ "$quiet" != "y" ] && log_begin_msg "Running /scripts/sfs-top" 
         run_scripts /scripts/sfs-top 
         [ "$quiet" != "y" ] && log_end_msg 
  
         modprobe af_packet 
  
         wait_for_udev 10 
  
         delay=${ROOTDELAY:-180} 
  
         do_sfsmount 
         while [ ${retry_nr} -lt ${delay} ] && [ ! -e ${rootmnt}${init} ]; do 
                 [ "$quiet" != "y" ] && log_begin_msg "Retrying sfs mount" 
                 /bin/sleep 1 
                 do_sfsmount 
                 retry_nr=$(( ${retry_nr} + 1 )) 
                 [ "$quiet" != "y" ] && log_end_msg 
         done 
  
         [ "$quiet" != "y" ] && log_begin_msg "Running /scripts/sfs-bottom" 
         run_scripts /scripts/sfs-bottom 
         [ "$quiet" != "y" ] && log_end_msg 
} 
  
  
Ramdisk引导系统过程就是init脚本了。经过检查硬件等各种准备工作之后，最后用run-init命令——相当于以前的pivot_root（）和chroot()——进入新系统，同时释放全部ramdisk占用的内存。 
一些需要注意的地方： 
         我们的目标系统的squashfs里面是不含/lib/modules的，需要一些modules在这儿加载。比如我的无盘节点需要挂载nfs存储，所以 加载了nfs。 
     modprobe -q loop 
     modprobe -q nfs 
  
修改此处，可以改变目标diskless node的根分区大小（不包含sqaushfs.img） 
mount -t tmpfs -o size=100M tmpfs /media/2 
  
重新生成ramdisk: 
# find . | cpio –H newc –o | gzip -9 > ../initrd.gz 
  
配置pxelinux 
#apt-get install syslinux-common 
#cp /usr/lib/syslinux/pxelinux.0 /data/tftpboot/pxelinux.0 
#mkdir /data/tftpboot/pxelinux.cfg && cd /data/tftpboot/pxelinux.cfg 
  
收集需要做无盘系统的服务器的eth0的mac地址 
如果mac地址为00:30:48:76:ab:4a，就编辑一个01-00-30-48-76-ab-4a文件。规则是将冒号变为减号，前面加01-。Pxelinux会优先查找这个文件。 
不同的diskless node写不同的配置文件，文件内容里面的sfsroot不一样，以此实现diskless node的定制化。 
如果都是一样的操作系统，只写一个default文件即可。 
  
编辑01-00-30-48-76-ab-4a 
DEFAULT linux 
LABEL   linux 
         kernel vmlinuz 
         append initrd=initrd.gz boot=sfs root=/dev/ram0 sfsroot=http://172.16.8.101/squashfs.img rootfstype=squashfs ip=dhcp rw ramdisk_size=524288 
  
  
         配置isc-dhcp-server 
  
安装isc-dhcp-server 
#apt-get install isc-dhcp-server 
  
编辑/etc/dhcp/dhcpd.conf: 
ddns-update-style none; 
log-facility local7; 
subnet 172.16.8.0 netmask 255.255.248.0 { 
   range 172.16.8.111 172.16.8.200; 
   option subnet-mask 255.255.248.0; 
   option broadcast-address 172.16.15.255; 
   option routers 172.16.8.1; 
   option domain-name-servers 202.106.0.20; 
   option domain-name "test.com"; 
   option host-name "squashnode"; 
   allow unknown-clients; 
   filename "pxelinux.0"; 
   next-server 172.16.8.101; 
} 
  
# invoke-rc.d isc-dhcp-server reload 
  
Dhcpd的filename和next-server指定了dhcp客户端得到ip后下一步的动作。 
  
配置tftpd  
# apt-get install tftpd-hpa xinetd 
  
编辑/etc/xinetd.d/tftp 
service tftp 
{ 
disable = no 
socket_type = dgram 
protocol = udp 
wait = yes 
user = root 
server = /usr/sbin/in.tftpd 
server_args = -s /data/tftpboot  
} 
  
# invoke-rc.d xinetd reload 
  
/data/tftpboot为tftpd的root目录。Diskless node的内核和ramdisk在这儿。 
  
  
    配置httpd 
#apt-get install nginx 
  
将nginx root目录配置到squashfs.img所在的位置即可。注意squashfs.img默认是600权限。要配置nginx进程的用户对其可读，否则diskless node的ramdisk引导过程会出错导致kernel panic。 
  
启动diskless node 
到这儿，我们的diskless server已经搭建完毕了。 
启动你的diskless node，进入BIOS修改启动顺序，改成优先PXE启动。 
Diskless server的dhcpd会分发给diskless node一个ip，并且告诉diskless node去哪儿干什么（dhcpd.conf的next-server和filename），然后diskless node通过tftp得到vmlinuz和initrd.gz，并且执行vmlinuz，vmlinuz最后会执行ramdisk，进入ramdisk文 件系统，且执行initrd的init脚本，以sfs形式启动，通过http下载squashfs.img,然后建真实系统根目录，mount上 squashfs，aufs，最后run-init切入到真实root，执行真实root里面实际上在squashfs.img里面的的/sbin /init，接下来就是一切正常的linux启动顺序啦。
