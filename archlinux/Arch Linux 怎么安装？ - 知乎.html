Arch的安装过程被人为复杂化了。它很可能是个过滤用户的阴谋：如果你按wiki装不上，那么就算给你台装好的arch，你用起来也必定很痛苦。
Linux老司机如果身边有别的arch机器，你只需要：
1. 用tar把那台机器的根目录整个打包（记得加--one-file-system参数，防止把/proc, /dev等打进来）。
2. 找个新一点的live usb启动机器（grml就挺好）。
3. 分区，mkfs。
4. 整个包解压到根目录。
5. 修改/etc/fstab里的dev或UUID。
6. 支持UEFI的机器上没必要grub，把vmlinuz-linux和initramfs-linux-fallback.img复制到EFI分区，efibootmgr加个启动项即可。精简版的initramfs可以等系统安完之后再生成。
7. 收工。
以上流程其实适用于各种发行版。

提供一个ArchLinux 网络安装的方法，为了尝试各种linux发行版，我没事就会倒腾各种版本，使用镜像安装比较麻烦，所以都尽量尝试netboot方式。Archlinux也提供了netboot安装方法。 安装步骤如下：

步骤一： 安装准备

1，下载ipxe.lkrn文件，放到一个非安装目录。

2，编辑grub.cfg, 增加一个启动项

menuentry "Network boot" {

linux16 /boot/ipxe.lkrn }

步骤二： 重启电脑，在grub菜单处选择Networkboot，该过程会获取DHPC IP，并下载启动文件。

步骤三： 在启动过后会进入archlinux的命令行环境，提供了基本的shell操作命令行，已经archlinux特有的安装命令。在更目录提供了install.txt。基本上按照install.txt步骤执行，则可以完成安装。

注意：因为是在线安装，所以选择一个下载速度快的镜像源非常重要，我使用阿里云的archlinux镜像源，基本上20分钟可以完成所需包的下载。 地址为：http://mirrors.aliyun.com/archlinux/iso/2017.05.01/arch/x86_64/

详细参考： https://www.archlinux.org/releng/netboot/

非常不建议看个人写的安装教程，因为很多教程写了后基本上不会再更新了，而 Archlinux 会时常发生一些大的变动。比如 2019 年 10 月的一个变动：

base 元包替代了同名的包组并且要求安装，需要手动干预升级 - Arch Linux​www.archlinuxcn.org

这个变动意味着所有 2019 年 10 月之前的安装教程如果没更新的话，那么就算你一步一步照着教程做也会安装失败（缺少内核）。

OK，我们可以来折腾折腾arch了，那么我来一步一步教你如何在vbox里面安装archlinux。


配置Vbox首先，在vbox里面新建一个VM，选择好Archlinux 64bit类型，然后一路下一步。

然后选中它，点击“设置”来对它进行一些简单的配置工作。


  1. 去掉软驱
  2. 如果你勾上“启用EFI”那么接下来的分区方式就要用GPT，引导方式EFI。（可自行参见archwiki）

在此，我们没有勾选启用EFI，经我自行测试发现vbox对EFI的模拟还不是很完善，系统有很大几率无法自动引导而是进入UEFI Shell，不过也无所谓，可以在这个shell中直接运行 *.efi文件来进入系统，真机上无此问题


然后，切换到“处理器”选项卡，如图配置

  1. 处理器分配自己拿捏吧，如果你觉得虚拟机处理太慢，按键有延迟等情况，可加大。
  2. 勾选这个选项，传递主板的电源管理到虚拟机，如果不支持的话会造成开机时出现
intel_rapl: no valid rapl domains found in package 0

告警，不过不影响进入系统。


最后，配置一下“显示”

  1. 显存自然是越大越好咯
  2. 3D加速需要安装显示驱动之后才能打开，在此勾不勾无所谓

到此，配置工作结束，选择存储，加载ISO光盘镜像后，点击“运行”。

  • 安装 ArchLinux

进入光盘系统之后，首先是对硬盘分区以及格式化。这个光盘系统本身自带了很多工具。可以从archwiki上进行了解。在此，我使用parted来分区。


1. 使用parted进入

2. mkpart的用法是 <主/次分区> <格式> <起> <止>

  • 使用msdos分区表（如果在之前的设置勾选了“启用EFI”则要使用GPT）
  • 创建一个主要分区，大小100M
  • 设置为boot分区
  • 创建一个交换分区，大小1G
  • 创建一个主要分区，大小为其余所有剩余空间
  • p 打印分区表，看看是不是我们想要的
  • q 退出


接下来便是格式化的工作了。lsblk 打印设备，可以看到刚才的分区对应的真实设备名为sda1~3

mk* 格式化设备

#/> mkfs.ext4 /dev/sda1
#/> mkfs.ext4 /dev/sda3
#/> mkswap /dev/sda2


挂载设备

一条一条的解释，

swapon 启用一个交换分区

mount 把sda3这个设备挂载到本系统的/mnt目录（本系统为我们正在运行的光盘系统）

cd /mnt 进入这个目录就相当于进入了sda3这个设备，也就是刚刚我们分区并格式化的这个设备

mkdir boot 建立一个boot目录

mount /dev/sda1 /mnt/boot 还记得第一个分区（100M）吗？挂载到boot上面

理论上，接下来我们就可以安装系统了。为了加速安装，我们添加一下网易的源。

使用命令

nano /etc/pacman.d/mirrorlist


  1. 用方向键调整光标，在合适的地方写上网易的源，然后使用ctrl+o保存修改，ctrl+x退出编辑。
  2. 使用pacman -Syy更新源以捕获刚刚添加的网易源

最后， 使用

pacstrap /mnt base base-devel

来安装系统到/mnt目录中去（也就是sda3分区上）

base-devel 是gcc g++ m4 make 等等编译打包相关的工具集我个人觉得是很必要的。

安装完成之后使用

genfstab -U /mnt >> /mnt/etc/fstab

意思是把/mnt目录下的所有挂载关系映射一份到etc下面以便于下次开机就会自动挂载，比如那个交换分区，那个sda1挂载到boot。

自此，系统就安装好了，接下来，我们聊聊配置系统

  • 配置ArchLinux

现在系统虽然已经安装上去了，但此时重启是进入不了系统的，因为还有引导程序没有安装。(注：如果使用GPT+EFI的方式此时应该是可以直接引导，如果不能可以试试安装systemd-boot)

首先切换到我们刚刚安装的那个系统的文件系统

arch-chroot /mnt /bin/bash

此时，根目录/ 变成了新安装系统的根，也就是原来位于/mnt的地方这样就再访问不到原光盘系统的文件了，使用bash登录root用户

下载grub

pacman -S grub

然后安装grub到sda设备上并生成配置文件

grub-install /dev/sda
grub-mkconfig -o /boot/grub/grub.cfg

这里不要选错了，不是sda1而是整个sda

OK，我很佩服你的耐心，能看到这里得话，你的系统就完全弄好了。你可以弹出ISO文件，重启一把


却看到了这个画面，方的一逼，以为自己安装失败，怎么进不去系统？其实仔细看看

如果不看后面那个警告，最前面localhost login: 这里就已经代表成功安装了，偏偏警告太过于显眼导致一些新手在这里心理崩盘重装。此时回车几下就好了。


怎么样，愉快的进入了系统！

使用root进入系统，前方有大量的配置等待着我们。

下面的工作如果有意折腾Arch的话，应该要自己写成一个shell脚本，以后重装的时候会大幅减少重装成本！而且，trust me，你一定会在折腾之初大量反复重装的。

  • 配置语言环境(编码)

这个工作非常重要，重要到它会关系到我们之后想要使用的zsh和zsh的主题oh-my-zsh会不会出现显示错乱和乱码。

使用nano编辑器编辑/etc/locale.gen，用ctrl+v向下翻页(ctrl+y向上翻)

放开en_US.UTF-8 这一行(默认这一行是被注释的)

这个环境是系统能正常工作的保证，当然它是英文的环境，如果你希望看到的中文的系统环境

再往下翻找到zh_CN.UTF-8 放开注释

保存退出

注意：en_US.UTF-8是必须要有的不管你是不是打算选择中文的环境除非你有能力解决你碰到的所有软件的兼容性问题。

然后使用

locale-gen

生成编码。然后使用

echo LANG=en_US.UTF-8 > /etc/locale.conf

配置系统优先语言环境为英文

  • 配置时间

设置时区

使用date命令，八成看到的是UTC+0时间，跟我们的时区相差8个小时

看一下/etc/localtime是否存在，如果存在可以使用rm删掉

然后使用软链接，把上海的timezone文件链接过来，使用如下命令

ln -s /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

最后使用date命令检查一下看看时间对不对。

  • 配置主机名

一旦配置了主机名，登录的用户就会显示出xxx@hostname的形式了。

echo Sjet > /etc/hostname

  • 创建用户

我们刚才一直是使用root裸奔，这对于linux而言是非常危险的，所以接下来的工作就是添加用户了

useradd -m -G wheel -s /bin/bash rowland

-m是创建的用户默认生成/home/xxx 的用户目录，下文用”~“来指代。

-G是追加到一个组里面去在此是wheel组，这个组是接下来我们要配置的拥有sudo权限的组

-s是使用哪种shell，在此呢，我们先使用bash，接下来我们也会改掉它

rowland是我的用户名了：）

然后创建密码

passwd rowland

到此用户创建完毕。

  • 为用户添加sudo权限

这个用户目前还没有sudo执行权限，所以暂时没有必要切换用户，还是使用root安装sudo。

pacman -S sudo

安装了sudo之后就使用

visudo

来配置那些用户可以使用sudo了。

visudo 使用的是vim的编辑环境，进入之后是命令模式，使用/privileget 回车是查找privilege关键字，如果要重复查找下一个可以直接打 / 之后回车。找到截图中的这一行的开头，然后按两下x放开这一行的注释。这一行代表wheel组用户可以使用sudo执行所有的命令，下面那一行的意思是说 wheel组用户可以不需要密码执行所有命令。大家可以按自己的需求来吧。改好了之后按两下ZZ保存退出（你可以使用shift+zz也可以打开caps lock之后按两下z）


OK，使用su rowland来切换用户，然后使用sudo reboot来测试一下。

重启之后可能上不了网，可以使用

sudo systemctl start dhcpcd

来启动dhcp服务进程。

sudo systemctl enable dhcpcd

来添加开机启动服务中去。

如果顺利重启，那可以接着往下看。cause 重头戏就要来了，那就是桌面DE环境，这个让人既爱又恨的linux桌面运行环境：）

  • 桌面

关于linux上面的桌面一般的架构都是采用的x，连接本地的xserver，调用x11接口来实现的，关于linux的桌面显示架构arch和wiki都有很完善的文档。在此就不多做介绍了。

所以，第一步安装x,xserver

sudo pacman -Syu xorg xorg-xinit xorg-utils dbus

然后就是安装桌面了，本来按照我一开始的打算是安装E的，但是试用了一段时间发现不是很稳定，所以在此还是决定以比较稳定的gnome为例吧，安装gnome

sudo pacman -Syu gnome gnome-extra

extra是gnome环境下的应用合集，如果你有洁癖可以只安装gnome。

安装完毕之后，拷贝一份xinit的初始化配置文件到用户目录下

cp /etc/X11/xinit/xinitrc ~/.xinitrc

接下来用nano编辑它，翻到最后，把twm &开始到末尾的信息全部注释掉，然后添加gnome的启动参数，如图（截图由于被压缩了导致把export后面的变量的下划线给压没了！注意了变量的下划线，文档：GNOME - ArchWiki）

export XDG_CURRENT_DESKTOP=GNOME-Classic:GNOME
export GNOME_SHELL_SESSION_MODE=classic
exec gnome-session --session=gnome-classic



接下来，激动人心的一刻来临了，使用命令

startx

启动桌面。

久违的桌面就这么赤果果的展现出来了！


  • 自动登录

每次手动startx肯定不爽吧，如果我们的确需要自动登录的话，就得使用dm登录管理器

sudo pacman -Syu gdm

安装gnome dm。

然后使用

sudo systemctl enable gdm

  • 安装驱动

进入了桌面但是发现图标大的爆表，窗口也无法拖大。因为我们还没有安装vbox的专用驱动vbox-guest-additions。

首先进入vbox官网下载最新的vbox extpackage扩展安装包，Linux用户下载guest additions updates


下载完成双击安装。

接下来到ArchLinux的桌面环境中，在安装驱动之前，先打开终端（点击gnome桌面左上角A标记图标，搜索terminal），然后使用

sudo pacman -S linux-headers

然后重启一把，现在你应该可以缩放你的vbox窗口了，试一试吧！

sudo pacman -S firefox

安装完成之后桌面左上角application里面搜索，打开浏览器。在performance中的content可以自己设置字体。

  • 安装配置emacs

emacs 是*nix系统最常见的编辑器之一，其优秀的设计哲学被*nix系统几乎所有的交互式编辑环境所采纳，包括terminal自身，nano编辑器，都或多 或少的吸收了emacs的精华。如果说vim是回合制rpg游戏的话，你总是要在策略模式和战斗模式间切换，让我觉得多少有点拖泥带水。那么emacs就 是即时rpg，拔刀猛砍，扒了猛干的直截了当。两种类型的游戏都能产生大量拥趸。在下也是一个坚定的emacs粉。

安装，无需多言

pacman -S emacs

安装完成之后直接在terminal运行

$> emacs

打开之后看到的是emacs窗口，别急，这会还没到如何使用emacs，叉掉它。

然后使用搜索引擎搜索“emacs elpa”关键词（elpa是emacs的插件包管理器，一开始是单独的插件，目前已经吸收集成到最新的emacs中了，elpa之于emacs就相当与pacman之于archlinux，apt-get之于ubuntu）

把如下两句话添加到~/.emacs中，这个文件不存在的话则需手动创建！

(add-to-list 'package-archives '("melpa" . "https://melpa.org/packages/"))
(add-to-list 'package-archives '("popkit" . "http://elpa.popkit.org/packages/"))

分别添加了两个插件包仓库的软件源，然后使用无窗口模式启动emacs

$> emacs -nw

使 用快捷键meta+x(在linux上alt键就是meta键！)进入命令行，输入package-list-packages（多次按tab键可以完成 命令提示和补全），如果我们能在5秒左右看到列表，说明我们刚才的配置已经生效了。每一列分别是“包名”，“版本”，“状态（可用\已安装）”，“源”， “描述”。

OK，走到这一步，大家应该觉得emacs比较棘手了吧，是时候教几个基本操作了。

emacs的操作一般都是快捷键实现的，大概会用到两个功能键 C键和M键，它在大多数linux上面对应ctrl和alt，mac电脑则是ctrl和esc键，当然你觉得不爽，它们都可以修改的。

当我说到“C-x 2”的时候，实际的操作是ctrl按住的情况下再按一下x 然后松开这两个键，最后按2

  • 纵向分屏（创建窗口）：C-x 2
  • 横向分屏：C-x 3
  • 切换窗口：C-x o
  • 关闭窗口：C-x 0
  • 下翻页：C-v
  • 上翻页：M-v
  • 保存：C-x C-s
  • 退出：C-x C-c
  • 光标到行首：C-a
  • 光标到行尾：C-e
  • 光标到上一行：C-p
  • 光标到下一行：C-n
  • 取消已输入的：C-g

好了，到此emacs的基本操作就这么多，是不是很简单呢？

回到包目录这里来，首先要安装的一个最重要的包就是自动补全提示插件——auto-complete

使用C-s 搜索auto-complete，重复C-s查找下一个。找到如图所示：


回车撞它。应该会在分屏出现它的信息，使用刚才教你的操作，切换到信息窗口，光标移动到Install关键词上面，回车。就可以进行安装了。是不是非常简单呢。退出，然后编辑~/.emacs配置文件，末尾追加

(require 'auto-complete)
(require 'auto-complete-config)
(ac-config-default)

保存退出之后再次使用emacs来编辑~/.emacs配置文件，尝试写一个“(req”看能否正确的提示出补全文法。


好了，已经可以提示了。auto-complete只是补全框架，所以要补全特定的编程语言，还不够。希望各位能够自己探索，勇于尝试，多问百度谷歌。多装装里面的插件，很多插件的安装信息窗口里面也给出了配置方式，多折腾折腾。俗话说师傅引进门，修为靠个人。
