                                                                               Debian9 PXE安装Centos6.5

实验环境为Debian_1做服务器，有双网卡，网段分别为192.168.10.1/24，192.168.20.1/24，Debian_2在10.0/24网段，Centos在20.0/24网段，使用PXE分别为Centos，Debian引导安装操作系统

先配置DHCP服务，分别下发10.0/24，20.0/24网段

subnet 192.168.10.0 netmask 255.255.255.0 {

  range 192.168.10.10 192.168.10.20;

  option routers 192.168.10.254;

  option broadcast-address 192.168.10.255;

  default-lease-time 600;

  max-lease-time 7200;

  next-server 192.168.10.1;

  filename "debian.0";

}

subnet 192.168.20.0 netmask 255.255.255.0 {

  range 192.168.20.10 192.168.20.20;

  option routers 192.168.20.254;

  option broadcast-address 192.168.20.255;

  default-lease-time 600;

  max-lease-time 7200;

  next-server 192.168.20.1;

  filename "centos6.5.0";

}

systemctl restart isc-dhcp-server

随后再来配置TFTP服务器，安装openbsd-inetd与tftpd-hpa

vim /etc/default/tftpd-hpa

TFTP_USERNAME="tftp"

TFTP_DIRECTORY="/tftpboot"

TFTP_ADDRESS="0.0.0.0:69"

TFTP_OPTIONS="--secure"

systemctl restart openbsd-inetd

systemctl restart tftpd-hpa

---------------------------------------------------------------------------------------------------------------------------------

PXE同时安装debian与Centos6.5的配置难度在于，需要让不同的引导程序获取不同的启动菜单，所以要实现这点，必须要了解到pxelinux.0文件寻找配置文件的顺序(pxelinux.cfg下的default)，所以这里先来说一下寻找的顺序：

  1. pxelinux.cfg/01-xx-xx-xx-xx-xx-xx                          ##01是为固定值
  2. pxelinux.cfg/C0A80A01 ##为客户端的IP地址
  3. pxelinux.cfg/C0A80A0
  4. pxelinux.cfg/C0A80A
  5. pxelinux.cfg/C0A80
  6. pxelinux.cfg/C0A8
  7. pxelinux.cfg/C0A
  8. pxelinux.cfg/C0
  9. pxelinux.cfg/C
  10. pxelinux.cfg/default                                             ##以上都没有才会寻找这个

了解过了这个之后便可以上手实验了，还是现将netboot.tar.gz的压缩包解压在/tftpboot目录下，该软件包的获取途径为

http://ftp.nl.debian.org/debian/dists/stretch/main/installer-i386/current/images/netboot/netboot.tar.gz，可以使用wget命令获取软件包

wget http://ftp.nl.debian.org/debian/dists/stretch/main/installer-i386/current/images/netboot/netboot.tar.gz

cd /tftpboot

tar -zxf netboot.tar.gz

mv netboot.tar.gz ~/

ls -l /tftpboot

debian-installer

ldlinux.c32 -> debian-installer/i386/boot-screens/ldlinux.c32

pxelinux.0 -> debian-installer/i386/pxelinux.0

pxelinux.cfg -> debian-installer/i386/pxelinux.cfg

version.info

mv /tftpboot/pxelinux.0 /tftpboot/debian.0

mv /tftpboot/pxelinux.cfg/default /tftpboot/pxelinux.cfg/01-00-0c-29-51-8e-6f    ##此处MAC地址为Debian客户机且一定要小写

到此为止便可以启动Debian客户机了，正常看到的画面应该是这样的：

 

接下来在配置Centos6.5的PXE引导，配置Centos的困难在于，Debian的apt源所提供的pxeliux.0，centos是运行不了的，所以你只能去centos的镜像中将他copy出来：

mkdir /tftpboot/centos6.5/                         ##创建centso6.5目录用于存放centos6.5的内核

mount /dev/sr0 /mnt/cdrom                      ##将centos6.5的镜像挂载至/mnt/cdrom

cd /mnt/cdrom/isolinux/                            ##进入/mnt/cdrom/isolinux复制文件

cp vmlinuz initrd.img /tftpboot/centos6.5  ##复制内核及镜像至centos6.5目录

---------------------------------------------------------------------------------------------------------------------------------

到此为止，镜像中需要的东西遍全copy出来了，剩下的还需要vesamenu.c32，menu.c32，pxelinux.0，这三个只能在centos6.5系统中安装了syslinux包后将他copy出来在上传到Debian系统，以下是在Centos6.5中的操作：

cd /usr/share/syslinux

cp pxelinux.0 menu.c32 vesamenu.c32 ~/

随后可以在自己的机器使用sftp连接上去然后将这三个文件copy下来，我这里用的是xshell的sftp：

然后在Debian_1的/home/user目录下可以看到这三个文件：

将这三个文件分别copy到/tftpboot目录下：

cp menu.c32 vesamenu.c32 /tftpboot

cp pxelinux.0 /tftpboot/centos6.5.0

进入/tftpboot/pxelinux.cfg/将isolinux.cfg更名为Centos6.5的MAC地址：

cd /tftpboot/pxelinux.cfg/

mv isolinux.cfg 01-00-0c-29-a7-6f-55

vim 01-00-0c-29-a7-6f-55

default vesamenu.c32

#prompt 1

timeout 600

display boot.msg

label linux

  menu label Centos6.5

  menu default

  kernel ../centos6.5/vmlinuz

  append initrd=../centos6.5/initrd.img

##这两行参数必须修改，因为我们将内核及镜像放在了centos6.5目录下

 到此为止便可以启动Centos客户机了，正常看到的画面应该是这样的：

---------------------------------------------------------------------------------------------------------------------------------

总结一下配置方法：

  1. 安装并配置DHCP服务器，添加filename,next-server，结尾注意分号
  2. 重启或启动isc-dhcp-server
  3. 安装openbsd-inetd，tftpd-hpa，并配置tftp，配置文件在/etc/default/tftpd-hpa
  4. 重启或启动openbsd-inetd和tftpd-hpa
  5. 下载Debian的netboot.tar.gz包
  6. 解压软件包至TFTP的根目录
  7. 将pxelinux.0更名为debian.0       ##只是为了方便区分
  8. 复制centos6.5的menu.c32 vesamenu.c32至TFTP根目录
  9. 复制centos6.5的pxelinux.0至TFTP根目录并更名为centos6.5.0 （注释1）
  10. 复制isolinux.cfg至TFTP根目录下的pxelinux.cfg并更名为01-centos’s MAC地址
  11. 在根目录新建Centos6.5文件夹，并将vmlinuz initrd.img放进去（注释2）
  12. 修改01-centos’s MAC，将内核及镜像地址，修改为../centos6.5/（注释3）

注释1：menu.c32 pxelinux.0 vesamenu.c32都在Centos6.5系统下/usr/share/syslinux

注释2：isolinux.cfg vmlinuz initrd.img都在Centos6.5镜像中的isolinux目录中

注释2：因为修改了相对路径，所以要在这里将相对路径改过来
