Ubuntu下利用Apache和WebSVN搭建SVN服务器

[日期：2012-06-10] 来源：Linux社区  作者：lalor [字体：大 中 小]

在vps上搭建svn服务器是一件很常见的工作。本文就简单介绍一下如何在Ubuntu下利用apache搭建一个svn服务器，并利用websvn进行在线浏览。本文假设读者已经安装好svn、php、apache以及apache的dav_module和dav_svn_module插件，并且假设用户名为linuxidc、网站域名为www.linuxidc.com、网站网页的存储位置为“/home/linuxidc/www/www.linuxidc.com/public_html”。

第一步，先建立相应的文件及文件夹。

 
1 mkdir ~/svn && cd ~/svn
2 mkdir repos
3 touch svn-auth
4 touch svn-policy

第二步，将文档库移动到上面创建的repos文件夹中。当然，也可以利用“svnadmin create”来创建一个新的版本库。值得注意的一点是，为了让apache能正常的处理文件，我们需要更改版本库的所有权。下面的命令就创建了一个名为test的版本库并更改其所有者为apache所管理的用户www-data。

 
1 cd ~/svn/repos
2 svnadmin create test
3 chown -R www-data:www-data test

第三步， 添加能够访问版本库的用户并为他们设置密码。下面的命令创建了两个用户linuxidc和alice，并分别为他们设置了密码。

 
1 cd ~/svn
2 htpasswd svn-auth alice
3 htpasswd svn-auth linuxidc

第四步，设置各个能够访问版本库的用户的权限。这里，我们使用svn的“path-based authorization”，不了解或者想了解更多的读者可以参考这里。例如，将svn-policy的内容改为如下使得linuxidc对所有的版本库都有读写权限，而alice仅对test版本库有读权限。

 
1 [/]
2 linuxidc = rw
3 [test:/]
4 alice = r

第五步，在www.linuxidc.com的apache配置文件中添加下列内容。

 
01 <Location /svn/>
02   Options Indexes FollowSymLinks
03   DAV svn
04   SVNParentPath /home/linuxidc/svn/repos
05   SVNListParentPath on
06   AuthzSVNAccessFile /home/linuxidc/svn/svn-policy
07   Require valid-user
08   AuthType Basic
09   AuthName "linuxidc.COM SVN Server Access"
10   AuthUserFile /home/linuxidc/svn/svn-auth
11 </Location>

至此，一个简单而灵活的svn服务器就搭建完成了。linuxidc可以使用svn并通过www.linuxidc.com/svn/test来对test这个版本库进行控制，linuxidc也可以直接访问www.linuxidc.com/svn来在线查看各个版本库最近一次的快照。但是，apache所提供的在线浏览的功能是很有限的。为了能够更好的在线浏览版本库，我们可以使用websvn。

第一步，我们前往http://www.websvn.info/download/下载最新版本的websvn并将其解压。假设解压出来的所有文件存放在“/home/linuxidc/www/www.linuxidc.com/public_html/websvn”中。

第二步，将websvn/include/distconfig.php拷贝一份并重命名为websvn/include/config.php

 
1 cp ~/www/www.linuxidc.com/public_html/websvn/include/distconfig.php ~/www/www.linuxidc.com/public_html/websvn/include/config.php

第三步，打开websvn/include/config.php并添加如下设置。大家也可以根据自己的需要进行其他设置。

  • $config->parentPath('/home/linuxidc/svn/repos');
  • $config->useAuthenticationFile('/home/linuxidc/svn/svn-policy');
  • $config->setBlockRobots();

第四步，在www.linuxidc.com的apache配置文件中添加下列内容。

 
1 <Location /websvn/>
2   Require valid-user
3   AuthType Basic
4   AuthName "linuxidc.COM WebSVN Viewer Access"
5   AuthUserFile /home/linuxidc/svn/svn-auth
6 </Location>

最后，重启apache服务器。

 
1 sudo /etc/init.d/apache2 restart

现在，我们就可以通过www.linuxidc.com/websvn进行在线浏览了。并且，我们前面设置的访问控制权限对其依然有效。
