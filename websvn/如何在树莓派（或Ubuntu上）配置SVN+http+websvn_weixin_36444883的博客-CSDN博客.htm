如何在树莓派（或Ubuntu上）配置SVN+http+websvn

本文介绍了在树莓派3B+上配置SVN库，用于管理项目组的文档和源码。该树莓派通过无线接入项目组办公室内的一个无线路由器，并在无线路由器内部做好端口映射，使得外部网络能够通过无线路由的端口号访问树莓派上的SVN库。本教程参考了CSDN上的链接：ubuntu 搭建 svn服务器，使用http方式访问。
同时本文还搭建WebSVN，便于授权的用户通过浏览器查看SVN库中的文件和提交记录。

为什么要用树莓派

树莓派的运行功率相较于一台PC机要小很多，而SVN服务器要求全天候运行，因此使用树莓派能起到节能的作用，并且SVN库的访问速度也满足使用要求。

为什么要使用无线路由的“虚拟服务器”功能（端口映射）

这是受限于项目的网络环境：联网设备需要先注册MAC地址，然后才能访问网络。也就是说，如果要将树莓派直接通过有线接入网络，必须去网络管理中心提交MAC地址，另外还要每年缴纳上网费用。项目组中正好有个公用的无线路由器，其MAC地址已经在网络管理中心完成注册，为了节约搭建成本，选择了将树莓派接入无线路由器中，然后开启无线路由器的“虚拟服务器”功能，使得项目组与无线路由器同在一个网络层的PC机，能够通过无线路由器的端口映射，访问树莓派。

为了保证无线路由器中树莓派IP地址的固定，需要将树莓派的MAC地址与无线网中的IP地址进行绑定，这样一来，即便是树莓派掉线，重新连上无线网后，也能保证IP地址不变。

例如无线路由器的IP地址是202.11.32.56，树莓派MAC地址绑定好的IP地址为192.168.1.102，那么端口映射设置如下

  • 3389，远程桌面端口， 可通过202.11.32.56:3389，采用windows的远程桌面与树莓派进行连接
  • 22，SSH协议端口，可通过202.11.32.56:6001，采用SSH协议与树莓派连接
  • 80，HTTP协议端口，可通过202.11.32.56::8080，采用HTTP协议与树莓派连接

为什么要使用HTTP访问SVN

svn服务是svn默认的访问形式，但是人们通常习惯于使用域名或者ip地址来访问一个项目，如果我们把svn服务器当做一个项目的话，使用http协议来访问svn服务更加符合我们的习惯。而且对于没有接触过svn的其他运维人员或者其他人员，http访问版本服务器要优于svn服务访问。

配置svn和http协议访问

树莓派操作系统：Raspbian

  1. 安装apache2和svn server
sudo apt-get install subversion
sudo apt-get install apache2
sudo apt-get install libapache2-mod-svn

验证：完成以上安装后，可以访问apache的主页：http://202.11.32.56:8080

  1. 创建svn版本库
sudo mkdir /home/pi/server
sudo svnadmin create /home/pi/server/repos

验证：此时能够在/home/pi/server/repos目录下看到svn版本库的所有文件夹以及配置文件，如：conf、db、format、hooks、locks、README.txt。

  1. 配置svn，编辑authz、passwd和svnserve.conf

authz用于配置svn的权限，passwd是保存用户名密码，为了方便统一管理，把这两个文件从/home/pi/server/repos/conf复制到/home/pi/server/文件夹下，然后进行编辑。

编辑/home/pi/server/passwd文件：

[users] 
admin = admin

编辑/home/pi/server/authz文件：

[groups] 
admin_group = admin

[/]
@admin_group = rw

编辑/home/pi/server/repos/conf/svnserve.conf文件

  • 设置匿名用户不可读：anon-access = none
  • 设置有权限的用户可写：authz-access = write
  • 设置密码文件路径：password-db = ../../passwd，注意这里是相对路径
  • 设置权限文件路径：authz-db = ../../authz，注意这里是相对路径
  1. 启动svn服务
sudo svnserve -d -r /home/pi/server

其中：-d 表示svnserver以“守护”进程模式运行； -r 指定文件系统的根位置（版本库的根目录），这样客户端不用输入全路径，就可以访问版本库：svn://localhost/repos

关闭svn服务： sudo killall svnserve

  1. 将svn配置到apache中通过HTTP方式访问

在/etc/apache2/mods-available/dav_svn.conf文件中配置如下内容：

<Location /svn>
    DAV svn
    SVNParentPath /home/pi/server           
    SVNAdvertiseV2Protocol Off
    AuthType Basic
    AuthName "Subversion Repository"
    AuthUserFile /home/pi/server/pwdfile
    AuthzSVNAccessFile /home/pi/server/authz
    Require valid-user 
</Location>

往/home/pi/server/pwdfile添加用户名和密码，这里的密码将会被加密。

sudo htpasswd -c /home/pi/server/pwdfile admin

其中：-c 表示新建pwdfile，如果新增用户名和密码，则需去掉-c
运行以上命令后，需输入两次密码，用户名和密码与/home/pi/server/passwd中一致。

  1. 重启apache服务
sudo /etc/init.d/apache2 restart

验证：可以通过http://202.11.32.56:8080/svn/repos/ 访问svn，期间需输入第5步配置的用户名和密码

配置websvn

WebSVN 将 Svbverion 的操作你的仓库的各种功能通过 Web 界面提供出来。通过它，我们可以看到任何给定版本的任何文件或者目录的日志，并且可看到所有文件改动、添加、删除的列表。我们同样可以查看两个版本间的差异来知道特定版本改动了什么。

  1. 安装websvn
    测试安装的websvn版本为2.3.3，从websvn官网下载。解压后将websvn文件夹移动至/var/www/html文件夹下。

  2. 配置websvn
    首先复制默认的配置文件，
    cd /var/www/html/websvn/include
    sudo cp distconfig.php config.php
    然后在/var/www/html/websvn/include/config.php的最末端添加

$config->setTemplatePath($locwebsvnreal.'/templates/calm/'); //set default template
$config->parentPath("/home/pi/server"); 
$config->setEnscriptPath("/usr/bin");  
$config->setSedPath("/bin");
$config->useEnscript();
$config->useAuthenticationFile('/home/pi/server/authz'); //set authentication file same as svn repository

如果需要直接添加svn版本库的路径，而不是采用$config->parentPath() 的方式添加父级目录，可以通过以下配置方式（两者选其一）：
$config->addRepository("repos name", "file:////home/pi/server/repos");

  1. 配置web与apache2连接，设置访问权限
    在/etc/apache2/mods-available/dav_svn.conf文件最末端增加如下内容：
<Location /websvn/>
    AuthType Basic
    AuthName "Subversion Repository"
    AuthUserFile /home/pi/server/pwdfile
    Require valid-user 
</Location>

  1. 重启apache
    sudo service apache2 restart
    或者
    sudo /etc/init.d/apache2 restart

  2. 打开浏览器，输入http://202.11.32.56:8080/websvn，即可打开websvn的登录界面，输入用户名和密码（与SVN库相同），可以查看文件与提交记录。

其他

  1. 配置开机自动运行svn服务:
    参考：svnserve开机启动
    经过测试如果经过HTTP转化，则已经可以保证开机自动运行，此时可以使用SVN服务。
    如果直接采用SVN协议访问，即svn://192.168.1.102/repos，那么就需要配置本步骤，保证开机启动SVN服务。

  2. 将svn版本库配置到移动硬盘上
    树莓派的USB端口目前无法直接驱动移动硬盘，需要通过辅助USB供电才能实现移动硬盘的挂载。然后将SVN库文件建立在移动硬盘上，参照本文的步骤即可完成配置。

  3. 访问权限问题
    出现Can’t open file ‘…/repos/db/txn-current-lock’: Permission denied的解决方案：
    sudo chmod -R 777 /home/pi/server/repos

  4. websvn配置时出现enscript问题
    出现"Unable to find “enscript” tool at location "/usr/bin/enscript"这样的问题，那么你需要安装enscript来修复这个问题。
    sudo apt-get install enscript

  5. 替换apache2的首页index.html文件，
    方便在http://202.11.32.56直接展示服务器上已经配置好的SVN库链接，需替换/var/www/html/index.html文件（该文件是默认的apache服务主页）内容如下：
    注意备份原主页为index-backup.html：sudo cp index.html index-backup.html
