本地

有一个tmux插件，是专门用于解决tmux剪贴板与系统剪贴板互通问题的，地址：tmux-plugins/tmux-yank

它的功能是：

  1. 使用tmux的copy模式（默认prefix+[）拷贝的内容
  2. 用鼠标拖动选定一段话，按y键拷贝的内容

都会自动同步到系统剪贴板

安装过程参照README就行，挺简单的。有的回答说要用到tpm（Tmux Plugin Manager），它是tmux的插件管理器，使用它装插件更方便。但和解决这个问题没有直接关系。


远程

yank插件解决了本地tmux的剪贴板同步，假如你通过ssh连接到远程服务器，tmux并非启动在本地，这个插件就不起作用了。这时候要祭出另外一个神器：wincent/clipper

简单的说：

  1. 安装clipper
  2. 配置clipper开机启动
  3. ssh命令增加一段。如：‘ssh -R localhost:8377:localhost:8377 root@remoteAddress’
  4. 更改服务器的tmux配置文件，增加一行。如：
bind-key -T copy-mode-vi 'y' send-keys -X copy-pipe-and-cancel "nc localhost 8377"

这样你使用tmux的复制命令，远程拷贝的内容就会自动同步到本机剪切板中。


详细的安装使用流程参见README，挺长的，但值得。clipper还支持配置到vim中，多折腾一下，你的所有开发环境的剪切板就都能互通了。

就是因为复制的不便性，我一直没有使用 Tmux，今天想彻底解决一下这个问题，先是搜索到了你的提问，发现回答中并没有比较满意的解决方案，因此继续死磕。在`~/.tmux.conf.loacl` 中无意间发现了这样一段配置代码

 # -- clipboard -----------------------------------------------------------------

 # in copy mode, copying selection also copies to the OS clipboard
 #   - true
 #   - false (default)
 # on macOS, this requires installing reattach-to-user-namespace, see README.md
 # on Linux, this requires xsel or xclip
 tmux_conf_copy_to_os_clipboard=true

(这段代码就是我用 iTerm2 相同的操作复制的)

于是接下来 `vim ~/.tmux/README.md` 查看，最后进入项目ChrisJohnsen/tmux-MacOSX-pasteboard 查看详细说明，其实解决这个过程只要3步，非常简单

  1. `brew install reattach-to-user-namespace`
  2. 在 `~/.tmux.conf.loacl`中增加`set-option -g default-command "reattach-to-user-namespace -l zsh"`
  3. 设置`tmux_conf_copy_to_os_clipboard=true`

重启tmux 尽情享用吧~

有时候跑程序忘记写输出文件，就想着怎么把输出的信息写到文件里。google搜到这个办法帮了大忙。

https://unix.stackexchange.com/questions/26548/write-all-tmux-scrollback-to-a-file​unix.stackexchange.com

1.5版本以上的tmux自带拷贝缓存信息的命令`capture-pane`，可以在.tmux.conf里加入下面这行

bind-key P command-prompt -p 'save history to filename:' -I '~/tmux.history' 'capture-pane -S -32768 ; save-buffer %1 ; delete-buffer'

自定义一个命令绑定到'P'，大家source-file之后试一下prefix+P就可以看到生成的文件了。
