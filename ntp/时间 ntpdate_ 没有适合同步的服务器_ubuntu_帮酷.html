回答 3:

我也有同样的问题，这就是我解决这个问题的方法。 我从上面到下给出了详细的答案。 你可能想跳到我的回答中，或者只是参考下面的截图来了解它。

首先安装 ( 显然你已经这么做了)

sudo apt-get install ntpdate

你需要按照如下所示配置带有服务器的ntp:

你至少应该在/etc/ntp.conf 配置 file: server 中设置以下参数

例如使用nano文本编辑器打开/etc/ntp.conf 文件：

sudo nano/etc/ntp.conf

定位服务器参数，如果它是空的，则如下所示：

server pool.ntp.org

但是在ubuntu中通常已经有些服务器是预先编写好

server 0.ubuntu.pool.ntp.org
server 1.ubuntu.pool.ntp.org
server 2.ubuntu.pool.ntp.org
server 3.ubuntu.pool.ntp.org

然后保存文件并重新启动ntpd服务:

sudo/etc/init.d/ntpd start

可以使用以下命令立即将系统时钟与NTP服务器同步：

sudo ntpdate pool.ntp.org

或者运行已经定义的服务器

sudo ntpdate 0.ubuntu.pool.ntp.org

或者使用服务器定义的其他服务器

一般

sudo ntpdate <one of the servername in/etc/ntp.conf>

你有机会得到这个错误，但是 NTP socket in use 如下所示：

在这种情况下

ntpdate -u pool.ntp.org

ntp守护进程运行时更新

或者停止守护进程，更新并重新启动它，如下所示：

sudo ntpdate pool.ntp.org
sudo service ntp stop
sudo ntpdate pool.ntp.org
sudo service ntp start

回答 4:

使用 htpdate

如果ntp传输在你的网络中被阻塞，请安装 htpdate。 它通过http协议同步时间。 根据手册页，精度将为 withing 0.5秒。

sudo apt-get install htpdate
sudo htpdate -a google.com

htpdate服务将在你安装软件包时启动。 如果有互联网连接，时间将立即更新。

即使我们面对这个问题。 但是我们通过点击这个命令来解决它：

ssh localhost"sudo apt-get install ntp ntpdate ; sudo service ntp stop; sudo ntpdate 0.ubuntu.pool.ntp.org; sudo service ntp start"

为此，你需要安装 openssh-server。

同时，删除ntp和 ntpdate ( sudo apt-get remove --purge ntp ntpdate ) 在触发上述 ssh 命令之前。
