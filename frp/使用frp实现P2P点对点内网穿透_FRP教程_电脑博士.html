嗯..............怎么填呢？还是举例说明吧，对照上面拓扑。

一、需求

frpc（客户端）与Visitor（访问端）间通过UDP传输（适合frpcs服务端小水管食用，Client于Visitor之间不受服务端带宽限制，主要看C与V的出口带宽和你的宽带质量。）

二、配置

客户端配置（拓扑图中右边“客户端B”）

[common]
server_addr = 1.1.1.8
server_port = 7000
token = xxxxxxxx

[p2p_rdp]
type = stcp
sk = 1234
local_ip = 127.0.0.1
local_port = 3389

重点

[common]
server_addr = 你的frpc服务器地址，上面用1.1.1.8举例
server_port = 你的frpc服务器端口，注意vps的iptables、firewalld或者vps管理端安全组放行该端口！  
token = 本端与服务器端认证密码，与服务器端配置一致！

[p2p_rdp]  名字稍微注意一下，下文中visitor端配置用到。
type = stcp   重点！！拓扑图中示例为xtcp，就稳定性和使用成功率而言目前xtcp不够成熟，优选stcp协议！
sk = client端和visitor端认证密码，注意两端要一致，与服务器无关
local_ip = 127.0.0.1
local_port = 你需要开放出去的端口，拓扑中以windows远程桌面为例

Visitor端配置（大部分配置教程也称之为“客户端”，我个人理解为访客端）

[common]
server_addr = 1.1.1.8
server_port = 7000
token = xxxxxxxx

[RDP_visitor]
type = stcp
role = visitor
server_name = p2p_rdp
sk = 1234
bind_addr = 127.0.0.1
bind_port = 53389

复习重点

[common]
server_addr = 1.1.1.8
server_port = 7000
token = xxxxxxxx

[RDP_visitor]  随意
type = stcp  与client端一致
role = visitor  好理解，这是“访客端”
server_name = p2p_rdp  注意上文中的提示，与client端配置一致
sk = 1234   还要再讲一次不？看笔记
bind_addr = 127.0.0.1  
bind_port = 53389 假如你配置的为该端口号

配置都正确后，在visitor端打开你的远程桌面连接（以Windows为例），计算机：127.0.0.1:53389(注意端口号)
