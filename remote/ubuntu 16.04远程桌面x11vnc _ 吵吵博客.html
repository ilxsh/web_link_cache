ubuntu 16.04远程桌面x11vnc
吵吵   2017-08-23  阅读：5,026

国防科技大学和工信部搞的一个基于ubuntu的linux系统kylin简直就是个笑话。

我是闲的蛋疼去安装和尝试了一下kylin,算了吧，劝各位还是放弃。

一会是网页打开要一分钟，一会是dpkg不知道被那个程序锁定，整体操作是又卡又错误多。

一度我怀疑我建立的虚拟机太差，跑不动这个系统。

几次尝试无果，最终还是决定干掉kylin ,装回到ubuntu官方的系统ubuntu 16.04 LTS，终于一切看起来正常了。

由于是服务器虚拟出来的虚拟机，所以远程桌面这个问题就必须要解决，你不能让我跑到机房接个显示器和键盘来用吧。

尝试了一下vnc4server，然后按照网上设置一大堆东西，搞半天没有搞好。

再尝试了一下x11vnc

先安装它 sudo apt-get install x11vnc

然后按照网上说的设置密码 x11vnc storepasswd

再带一堆的参数执行 x11vnc -display :0 -auth /var/run/lightdm/root/:0 -forever -bg -o /var/log/x11vnc.log -rfbauth /etc/x11vnc.pass -rfbport 5900

再去试一下，压根就不行好不！

在Dash里面搜索看到了X11VNCServer这个图标，咦，有图形版的，干嘛那么费劲去设置啊！

打开X11VNCServer，第一步设置端口5900后点击“ok”。第二步勾选“Accept Connections”，再输入Password，点击“OK”就搞定了，简单吧！

客户端这边就要去下载VNC Viewer这个软件，绿色版的一个exe就搞定，填写 IP:5900,点击连接就搞定了！

之后就可以放心远程了！

别急，由于我用的主机是服务器，没有显卡的，所以3D加速的任务就落到了cpu上，compiz这个进程一度占据30%的cpu,而且窗口移动起来也一卡一卡的，想办法关了compiz，结果发现不停，人家unity是依赖它的，那咋整？

装回原来的老界面吧！

sudo apt-get install gnome-session-flashback

然后在开机用户登录的时候选择点击图标选择 GNOME Flashback (Metacity).

我们就回到原来清爽的界面了，虽然不炫酷，但是很流畅！
