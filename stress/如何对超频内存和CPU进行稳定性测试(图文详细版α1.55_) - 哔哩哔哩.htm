如何对超频内存和CPU进行稳定性测试(图文详细版α1.55*)

数码
neil78
关注

- 前言 -

说明，本文偏重内存相关测试。笔者只是普通的超频玩家，对软件啥的还算熟悉（并不是程序相关专业毕业的娃），

这些软件的使用也是在研究中（我说的“研究”是实战方面的啊，记住），

有大大看出问题了请评价，我立马修正啊。

【还有，给英文小白的话，要去学点英语啊。下面也不可能从零说起啊，简单的提下。】

-压力测试 准备-

01. 注意好主机系统的散热状况，有些人的机箱是“焖罐”，建议把机箱前盖或侧盖打开，电源插板要买好的（请插到有地线的插座上），保持室内通风散热。

02. 在BIOS里面弄好，一般而言能稳定 CPU超频设定。举例来说，2950x 一般开 PBO，cpu 频率 4.1GHz，没必要非要改成 cpu core 电压 1.45v，超频 4.3GHz 去跑压力测试，特别是对于 LinX。

02.1* 其他关于二代线程撕裂者 cpu 的帖子，

https://www.overclock.net/forum/10-amd-cpus/1705286-ryzen-threadripper-2990wx-2970wx-2950x-2920x-owners-club.html

03. 正常启动 windows系统，禁用除了驱动程序以外的启动项目。可用 CCleaner 或者 SysinternalsSuite 里的 Autoruns。

04. 测试开始先关闭除了除了驱动程序以外的所有程序。当然，系统信息查看软件得开着。

05. 超频失败后。先尝试清除 CMOS设置，然后再重新设置BIOS。

清除 CMOS，（CLEAR CMOS，有些主板直接有一个专门的按钮，比较方便）

其他方法：关闭电脑，断开电源。05a. 小心地取出主板电池，拿螺丝刀把电池底座的正负级短接 1分钟，再装回电池；05b. 找到 CLRTC跳线，将 CLRTC 跳线帽由【1 - 2】(默认值)改为【2 - 3】约五至十秒钟。 二选一就行。

aida64

A. 表示此软件很多人都熟悉啊，从看硬件、看温度到各种测试应有尽有。

打开选项卡 - 工具 - 系统稳定性测试，右下角 Preferences，暂时只改 General 和 Temperatures 就行。

General，即常用选项，勾选 Use AVX when available。

Temperatures，温度传感器，在下面各种易辨别的颜色带上，选择以记录各种设备的温度，这里选的是 CPU，VRM，Chipset，SSD，Motherboard 。

Temperatures Graph，可以设置最大值(Max.value)，最小值(Min.value)，线条粗细(Thickness)。

弄好后确定，超频 CPU可以用 Stress FPU（CPU浮点运算测试），达到 CPU最大温度。

A1 . CPU超频稳定性 判断依据: 30分钟 Stress FPU 不出现蓝屏、黑屏、死机，CPU温度、CPU core电压超过自己理想的设定。（说明，不同的主板，cpu core电压不同；不要把不同主板的 cpu core电压进行比较）。

A2. 内存超频稳定性 判断依据: Stress system memory，15分钟没出现任何错误（包括死机、蓝屏、黑屏）。这个测试可发现一些内存的小问题，到15分钟就得换软件了。如果发现错误，测试会自动停止。


HWiNFO64

B. HWiNFO64 也比较推荐，也可以查看各种系统信息、电压等等，相当于一个不包含压力测试功能的 aida64，很多玩机大佬也爱用。

B1. 这里说个问题，华硕新主板（x470，x399）的 BIOS 如果不包含 ASUS WMI (华硕 WMI 传感器)，会导致主板温度控制、显示错误，结果在使用 HWiNFO64或 aida64的时候，会发生主机黑屏死机（无法强制关机，只能断电解决，亲身经历）。

B1.2 请各位玩家注意更新 BIOS ，如果真有难以解决的温度控制错误，只能看看 overclock.net 有没有大神制作包含 ASUS WMI的 BIOS，因为官方只能慢慢更新。

B2. 如果你使用Linux系统，也刷好了包含 ASUS WMI的 BIOS ，可以安装 Linux ASUS WMI Sensor driver，https://github.com/electrified/asus-wmi-sensors ，从这个地址的内容也可以看出华硕已经放出包含 ASUS WMI-BIOS的主板（笔者无法做统计啊）。

B2. 在 HWiNFO64 设置 - Safety - CPU Clock Measurement 里面，取消勾选 "Periodic polling"， 这个选项可能会导致 cpu / 内存频率显示错误。

来源，https://www.overclock.net/forum/26290421-post1.html

B3. 取消勾选 "HPET", "HPET" 还是关了好，其他选项不需改动，否则可能发生之前说的“黑屏问题”。


其他 系统信息查看软件

C. 有像 windows系统自带 msinfo32.exe， CPU-Z 、GPU-Z，CoreTemp 比较熟悉的；

也有像 SIV (System Information Viewer, http://www.rh-software.com/) , Statuscore 

（http://www.dannotech.com/statuscore/ ，Handy app to test individual/multiple real/SMT cores for clocks/temps/etc)，不是很熟悉的。

来源，https://www.overclock.net/forum/26290421-post1.html

C1. CPU-Z 建议使用主板 OEM版本的，即华硕主板，可用 ROG CPU-Z。据林总的视频说，CPU-Z 的版本不同显示的 CPU Core Voltage 也不同，因此新主板建议使用最新版的 CPU-Z 。


Prime95

D. 内容参考来源，http://overclocking.guide/stability-testing-with-prime-95/

 下载软件后，点击 “Just Stress Testing” 开始，

在测试选项中可以用各种 FFT sizes 组合以测试系统的稳定性：

（首先需要打开 aida64 或者 CoreTemp等软件，以查看电脑主机各设备的温度 ）

D1. 第一个选项，Small FFTs，（CPU浮点运算测试），使达到 CPU最大温度，和 aida64的Stress FPU类似，但不能完全保证超频 CPU的稳定，查看 D2。

D2. 在 Custom选项里，填入  Min FFT 1344K，Max FFT 1344K, 勾选 "Run FFTs in-place"，

把 "Time to run each FFT size" 设置为 15分钟，测试时长 30分钟，还要一直检查 CPU温度，看是否达到了 CPU最高温度。一般情况下，这个测试就能保证超频 CPU的稳定。

D2.1 CPU超频稳定性 判断依据:   30分钟 Stress FPU 不出现蓝屏、黑屏、死机，程序报错(Hardware Failure)，CPU温度、CPU core电压超过自己理想的设定。

D3. 备选的测试法，放大招，需要测试 21小时，Prime95 会按着下面的 FFT sizes 序列进行测试:

448k, 8k, 512k, 12k, 576k, 18k, 672k, 21k, 768k, 25k, 864k, 32k, 960k, 36k, 1120k, 48k, 1200k, 60k, 1344k, 72k, 1536k, 84k, 1728k, 100k, 1920k, 120k, 2240k, 140k, 2400k, 160k, 2688k, 192k, 2880k, 224k, 3200k, 256k, 3456k, 288k, 3840k, 336k, 400k, 480k, 10k, 560k, 16k, 640k, 20k, 720k, 24k, 800k, 28k, 896k, 35k, 1024k, 40k, 1152k, 50k, 1280k, 64k, 1440k, 80k, 1600k, 96k, 1792k, 112k, 2048k, 128k, 2304k, 144k, 2560k, 168k, 2800k, 200k, 3072k, 240k, 3360k, 280k, 3584k, 320k, 4000k, 384k, 4096k

你需要按着下图进行设置，

D4. The correct torture test settings are important to test the right things of your system. These are the most important FFT sizes, (正确的 “压力测试设定” 对测试系统是很重要的，以下是几个最常用的 FFT sizes)

【填的时候，Min FFT 和 Max FFT 都要填一样的 】

1344K: High load on the core and the core voltage （对 CPU内核和 CPU电压形成高负载测试）

8K: Maximum heat production to test your cooling solution（对系统产生最大热量以测试主机的冷却系统）

800K: Test RAM stability （测试内存稳定性），这里 使用内存要填可用内存的 80%至 90%

D4.1 内存超频稳定性 判断依据: 30分钟没出现任何错误（包括死机、蓝屏、黑屏）。如果发现错误，测试会自动停止。

864K: Test the whole system （可测试整个系统的稳定性），同样，使用内存要填可用内存的 80%至 90%

D4.2 系统超频稳定性 判断依据: 30分钟没出现任何错误，包括死机、蓝屏、黑屏，程序报错(Hardware Failure)。如果发现错误，测试会自动停止。

D5. 有其他经验啥的可评论啊。prime95 进阶使用相关的链接，https://www.reddit.com/r/overclocking/comments/a814aj/psa_dont_use_prime95_until_youve_read_this/

D6. 新版 Prime95 已经添加新的压力测试 Large FFTs，可用于测试内存，测试时长依然是 30分钟以上。


ＴＭ５ v3 config (for AMD Ryzen)

E. testmem5，俄国某大佬制作的内存测试软件，地址 http://testmem.tz.ru/testmem5.htm ，看不懂俄语千万不要打开，这里主要是讲1usmus大大制作的 v3 config（V3配置）

文件来源 https://www.overclock.net/forum/27937684-post4314.html

E0. 使用前，鼠标右键点击文件 TM5.exe，在“文件属性 - 兼容性”中给程序添加“管理员权限”，添加完成后，即可直接双击程序运行。


E1. 和 Memtest 一样，完成测试的时间，完全取决于主机安装内存的多少，64GB内存笔者测需要 40分钟左右，16GB，朋友的测试 10分钟左右。

E1.2 内存超频稳定性 判断依据: 直到测试完成，都没出现任何错误（包括死机、蓝屏、黑屏）。如果报错测试也会继续跑，直到圈数跑完。

E2. 使用注意，首次启动程序后，先把 TM5\bin\ 下的 Cfg.link文件删除，否则  v3 config 无法起作用，就首次启动程序啊。

E3. 有 2 个版本，一个给 Ryzen 桌面级 CPU，一个给 HEDT级 CPU （四通道，线程撕裂者），双击打开程序后，TM5 会按照特定的序列（Sequence）进行内存测试，跑特定的圈数（Cycles），

比如 配置文件中

Test Sequence=6,12,2,10,5,1,4,3,0,13,9,14,7,8,1,11,15，

Cycles=3 （跑3圈）

E4. 如果报错测试也会继续跑，直到圈数跑完。

E5. 不要修改配置文里面的内容。

E6*. 快速测试法：可以只跑一圈，然后用别的软件测试 ，如 RunMemtestPro，Prime95 的Large FFTs 压力测试。


 HCI memtest

F1. memtest 目前最可靠的内存稳定性测试软件，可搜索 RunMemtestPro。

官网 https://hcidesign.com/memtest/

F2. RunMemtestPro 是玩家自制的一个程序，用来更方便地运行 Memtest ，相当于一个运行脚本，可以帮你计算，截图、记录日志等等。

F2. 注意图中的 截图阈值 属于翻译错误啊，这里是 Completion Threshold Percentag，测试内存百分比。比如，内存稳定性测试 一般要过测 Memtest 200%，这里就填 200。

F3. 内存超频稳定性 判断依据: 直到 200%的测试完成，都没出现任何错误（包括死机、蓝屏、黑屏）。如果报错测试也可能会继续跑，直到测试跑完。

LinX

G1. LinX 是一款基于 Intel Linpack数学核心库稳定性测试软件

具体看查看，https://software.intel.com/es-es/blogs/2012/02/23/intelr-optimized-linpack-benchmark

G2. 新版 LinX地址，https://hwtips.tistory.com/1611

LinX 新版制作者的建议，别用 LinX 去测试内存稳定性。


G3. 一般测试CPU 超频稳定性跑 20圈，设置 4 - 8GB 内存，Problem Size也会跟着变动的。需要说明的是，LinX 的压力测试比 prime95的压力还大，请务必注意主机散热和超频设定。上图中我的测试， 2950x + ZE 只开PBO 全程 68℃，以 3.75GHz 的频率跑下來了，由于安装了64GB内存，所以设置了16GB，千万不要设置ALL （全部内存）。

G4. CPU超频稳定性 判断依据: 直到测试完成，都没出现任何错误（包括死机、蓝屏、黑屏）。并且，性能测试结果（GFLOPS）的差距越小越好。

G5. 提供2个已修复英文的 LinX 的下载，把 LinX.lng 留着，下次更新直接用它覆盖了新版里面的 LinX.lng 文件就行。



Windows内存诊断工具

H1. 系统自带的工具，对于某些内存错误还是检测的出来的，但速度或者可靠性比其他软件差一点。

H2. 对于不超频的内存条，其他类型的内存条，用来检测内存是否出现物理性的损坏等，比较有用。


 - 总结 -

i1. 内存的稳定性测试：

{可靠性}，aida64 内存稳定测 15分钟 < ＴＭ５ 0.12 v3 config < memtest 

{检测速度}，aida64 内存稳定测 15分钟 > ＴＭ５ 0.12 v3 config > memtest

一般可以先用 ＴＭ５ 0.12 v3配置测试，接着用 memtest 测试保证稳定性。

 i2. 实测法，用比较吃资源的大型单机游戏实测，但先要用 memtest 测过。

 i3.  CPU稳定性测试：

存在测试方法的不同，一般可以先 aida64 FPU 压力测试 30分钟，然后用 prime95 或 LinX。
