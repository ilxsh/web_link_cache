No idea about the problem you are asking about but I have two comments: first you have manually set up xrdp.repo and you should probably use the "official" way to access this repo. Since you already have it installed, just do yum install nux-dextop-release then remove your handcrafted xrdp.repo file. This way, if nux decides to push out an update to his nux-dextop repo definition, you will pick it up by updating nux-dextop-release as part of a yum update.

Second, and possibly negating the need for nux--dextop at all, EPEL is now shipping an xrdp package and it is a much higher version number than the one in nux. You might want to try using that one instead - if you don't have EPEL installed then use yum --enablerepo=extras install epel-release to get it. A subsequent yum update should just update nux's xrdp 0.6.1 to EPEL's 0.9.0 and then you may be able to remove nux-dextop altogether (unless you use it for multimedia packages etc).
