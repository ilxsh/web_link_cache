generate data file to be used to generate mcs file

Hi,

 

I would like to configure the ISF of 3S400AN with a bit file and some user data. I know that IMPACT (12.4 and 11.4) will not work so I am using promgen to generate an mcs file.

However, I am not sure about the format of the file I would like to add as user data.

 

I used .mem file as it mentioned in the data2mem.pdf but the final mcs interpret these values as ascii codes:

 

E.g. my "isf.mem" file is

@06FC00 E0
@06FC01 01
@06FC02 23
@06FC03 FF
@06FC04 45
@06FC05 21
@06FC06 00

 

the result mcs file has the line as

 

:020000040006F4
:10FC0000403036464330302045300D0A40303646CD
:10FC10004330312030310D0A4030364643303220F7

 

According to Chapman's explanation on mcs file (http://forums.xilinx.com/t5/New-Users-Forum/Required-mcs-data-format-information-for-FPGA-configuration/m-p/259270#M1569), the addresses are correct. However, the values are not

correct and as I mentioned earlier they are the Ascii codes of the values.

 

I would appreciate it if you could let me know how I can add a user data to mcs to configure the ISF with both bit file and the user data.

 

Regards,

 

NS

Re: generate data file to be used to generate mcs file

One thing I just found is that the addressing I was using in the promgen was not correct. I am using the default addressing mode and I used this promgen command to put the data at the address=0x08_0000:

 

promgen -p mcs -c FF -o xxx -spi -u 0 xxx.bit -data_file up 080000 isf_init_file.mem

 

which is not correct because in default mode the addressing is not continuous and I had to write:

 

promgen -p mcs -c FF -o xxx -spi -u 0 xxx.bit -data_file up 42000 isf_init_file.mem

 

See: http://www.xilinx.com/support/answers/42557.html

 

Inside my vhdl code, however, I should use the address: 0x08_0000 inorder to access the data. Now I can read the data the promgen is written at the end of the mcs file.

 

My other question still remains: how I can prepare data file for the promgen command line so that it really add the proper data (not its ascii code) to the mcs file.

 

Thanks,

 

NS

Re: generate data file to be used to generate mcs file

Here what I found, the correct promgen command is:

 

> promgen -w -p mcs -c FF -o xxx -spi -u 0 xxx.bit -bd isf_init_file_v0.mem start 399F0

 

I was missing word "start"! The promgen does not take into account the address you mention in your mem file. So my mem file looks like this:

 

@06FC00                                      <- I just write this address for my future reference
0E      // FPGA_MJR_REV
00      // FPGA_MNR_REV
00      // FPGA_INC_REV
00      // FPGA_PRGM_MODEL
00      // FPGA_CUR_LOC

.....

 

Cheers,
