          fail to create MCS file with vivado

          I am currently trying to create a mcs file using a  ISE projects' bit file in vivado environment. After I typed the command in the tcl console, it occurs error 

          [Writecfgmem 68-2] Bitfile a7_board.bit uses conflicting configuration settings for both BPI and SPI configuration.  Problem settings: SPI_buswidth=1, BPI_sync_mode=Type2.

          The project is using BPI so I don't understand why I need to adjust the SPI buswidth and I can not find a way to set the BPI_sync_mode under the ISE environment.

          I know its easier to just use ISE tool to generate this mcs file, but this is my current task that I really needs to overcome this problem, any help is appreciated.
