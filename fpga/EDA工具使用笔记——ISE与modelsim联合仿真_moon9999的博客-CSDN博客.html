ISE版本14.4，modelsim版本为se-64 10.2c

1.保证ISE和modelsim已经正确安装

2.在开始菜单中中，找到Xilinx Design Tools--EDK---Tools--Compile Simulation Libraries，点击来生成ise器件的库文件


2.选择modelsim SE 64-bit（根据所安装版本正确选择），下面选择modelsim的安装位置，一直选择到底层的文件夹只有一路next等待其生成



3.等待一段时间后，生成结束，查看报告，如有error的话可暂时忽略，点击finish



4.添加ISE EDK的环境变量

5.找到生成modelsim.ini文件

本机中位置位于"F:\Xilinx\14.4\ISE_DS\EDK"中，右键打开modelsim.ini文件，复制如下内容

6.复制入modelsim该文件下

在modelsim文件夹“C:\modeltech64_10.2c”中找到modelsim.ini，打开将刚刚复制的内容粘贴进去，保存文件


7.打开modelsim可以看到库已经添加进去了，关闭软件


8.建立工程编写代码和testbench

为验证是否可以使用，我选择调用了一个Clocking Wizard IP核

代码如下

`timescale 1ns / 1ps
module testbench;

	// Inputs
	reg clk;
	reg btn;

	// Outputs
	wire gpio1;

	// Instantiate the Unit Under Test (UUT)
	work uut (
		.clk(clk), 
		.btn(btn), 
		.gpio1(gpio1)
	);

	initial begin
		clk = 0;
		forever  #5 clk = !clk;
	end
	
	initial begin
		btn = 0;
		#4 btn= !btn;
		#4 btn= !btn;
	end
      
endmodule
主程序为12M分频为25M的
module work(
    input clk,
    input btn,
	 output gpio1
    );
wire rst;
assign rst = btn;

wire clk_25;
pll u0(
  .CLK_IN1(clk)
  ,.CLK_OUT1(clk_25)
 );

assign gpio1 = clk_25;
endmodule

9.进行如下设置

双击上图中的器件，在simulator出选择Modelsim-SE Verilog


Edit--Preferences--Tntegrated Tools,设置仿真工具modelsim路径


9.配置完成，点击Simulate Behavioral Model，会打开modelsim


10.此时若发现无仿真波形，此时点击View--Transcript打开脚本，会发下如下报错

Failed to access library 'rtl_work' at "rtl_work

应该是因为电脑上同时安装了quartus ii的原因

此时点开modelsim文件夹中的modelsim.ini文件，用“;”注释掉work = rtl_work，保存文件


11.关闭modelsim，重新从ISE打开，即可观察波形
