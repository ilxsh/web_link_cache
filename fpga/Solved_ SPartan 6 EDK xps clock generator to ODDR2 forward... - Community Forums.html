SPartan 6 EDK xps clock generator to ODDR2 forwarding to external output port

Hello,

 

I am creating clock output at 27Mhz oit of 100 Mhz input clock. 

I am designing embedded design with xilinx platrform studio with Microblaze core. 

But I met a problem while I connected  clokc generator with ODDR2.

I got these errors. 

 

ERROR:PhysDesignRules:1939 - Issue with pin connections and/or configuration on
block:<clock_buffer_0/clock_buffer_0/USER_LOGIC_I/ODDR2_inst>:<OLOGIC2_OUTFF>
. The OUTFF Flip-flop SRTYPE_OQ mode must be ASYNC in DDR mode with
DDR_ALIGNMENT mode of C0 or C1.
ERROR:Pack:1642 - Errors in physical DRC.
ERROR:Xflow - Program map returned error code 2. Aborting flow execution...

 

But in the case of connection, I put the ddr mode : ASYNC in user logic file. and updated IP core at xps. 

Why system saying that DDR mode SYNC, while I set ASYNC. ?  

Bello, is my custom IP at EDK. 

PLease help me.

Re: SPartan 6 EDK xps clock generator to ODDR2 forwarding to external output port

Jump to solution

Please check the implemented design and check the property of ODDR2 instantiation.

 

Ideally the instantiation should look like this.

 

ODDR2 Instantiation:

ODDR2 #( 

 
      .DDR_ALIGNMENT("NONE"), // Sets output alignment to "NONE", "C0" or "C1"
 
      .INIT(1'b0),    // Sets initial state of the Q output to 1'b0 or 1'b1
 
      .SRTYPE("SYNC") // Specifies "SYNC" or "ASYNC" set/reset
 
   ) clock_forward_inst (
 
      .Q(<output_clock>),     // 1-bit DDR output data
 
      .C0(<internal_clock>),  // 1-bit clock input
 
      .C1(~<internal_clock>), // 1-bit clock input
 
      .CE(<stop_clock>),      // 1-bit clock enable input
 
      .D0(1'b0), // 1-bit data input (associated with C0)
 
      .D1(1'b1), // 1-bit data input (associated with C1)
 
      .R(<hold_clock_low>),   // 1-bit reset input
 
      .S(<hold_clock_high>)   // 1-bit set input
 
   );
