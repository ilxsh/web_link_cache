Re: Clock forwarding

You need to be a bit more specific with your request.   A 10G signal means 10 Gbps and the only way to realize this would be with a MGT (multi-gigabit transceiver) and this would be a serial data stream that a receiver would use CDR to receive.  So there is no need for 10G (GHz) clock.

 

Note: If it was possible to generate a 10 GHz clock then data would like be specified to operate at 20 Gbps.

Re: Clock forwarding

Assuming you have a clock on a global clock net (driven by a BUFG), the proper technique is to instantiate an ODDR component to drive the output pin.  Details depend on what fpga family you are using.  Example for Spartan6:

 

  clk40_inv <= not clk40;          
  u_clk40_oddr2 : ODDR2            
    generic map                    
    ( DDR_ALIGNMENT  =>  "NONE"    
    , INIT           =>  '0'       
    , SRTYPE         =>  "SYNC"    
    )                              
    port map                       
    ( Q              =>  clk40_fwd 
    , C0             =>  clk40     
    , C1             =>  clk40_inv 
    , CE             =>  '1'       
    , D0             =>  '1'       
    , D1             =>  '0'       
    , R              =>  '0'       
    , S              =>  '0'       
    );
