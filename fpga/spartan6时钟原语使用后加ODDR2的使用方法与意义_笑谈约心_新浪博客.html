spartan6时钟原语使用后加ODDR2的使用方法与意义

(2014-04-30 14:14:53)
转载▼
标签：

xilinx

oddr2使用方法

时钟原语使用

错误解决

it

 分类： FPGA

    最近在做公司项目时，之前直接调用时钟原语即可完成的代码现在居然出问题了，根据错误提示，需要加一个ODDR2的东西，网上猛查资料，才发现在SPARTAN6的设计中，由时钟管理IP分出来的时钟线是不能直连到一些FPGA引脚的，错误是   

   ERROR:Place:1205 - This design contains a global buffer instance,
   , driving the net, , that is driving the
   following (first 30) non-clock load pins off chip.
   < PIN: CLKFX_OUT.O; >
   This design practice, in Spartan-6, can lead to an unroutable situation due
   to limitations in the global routing. If the design does route there may be
   excessive delay or skew on this net. It is recommended to use a Clock
   Forwarding technique to create a reliable and repeatable low skew solution:
   instantiate an ODDR2 component; tie the .D0 pin to Logic1; tie the .D1 pin to
   Logic0; tie the clock net to be forwarded to .C0; tie the inverted clock to
   .C1. If you wish to override this recommendation, you may use the
   CLOCK_DEDICATED_ROUTE constraint (given below) in the .ucf file to demote
   this message to a WARNING and allow your design to continue. Although the net
   may still not route, you will be able to analyze the failure in FPGA_Editor.
   < PIN "CLKFX_BUFG_inst.O" CLOCK_DEDICATED_ROUTE = FALSE; >

 

   翻译过来就是需要加一个ODDR2，该模块是一个IO BUFFER，你将ODDR的D0连到高电平，将D1连到低电平，将时钟连到C0，将时钟取反连到C1然后将ODDR的输出再连到FPGA的引脚上去。关于ODDR2的具体细节，可以参考Xilinx的技术文档：Spartan-6 Libraries Guide for HDL Designs。

 

  调用后既是

   ODDR2 #(
      .DDR_ALIGNMENT("NONE"), // Sets output alignment to "NONE", "C0" or "C1"
      .INIT(1'b0),    // Sets initial state of the Q output to 1'b0 or 1'b1
      .SRTYPE("SYNC") // Specifies "SYNC" or "ASYNC" set/reset
   ) ODDR2_inst2 (
      .Q(CLKFX_OUT),   // 1-bit DDR output data
      .C0(CLKFX_OUT1),   // 1-bit clock input
      .C1(~CLKFX_OUT1),   // 1-bit clock input
      .CE(1'b1), // 1-bit clock enable input
      .D0(1'b1), // 1-bit data input (associated with C0)
      .D1(1'b0), // 1-bit data input (associated with C1)
      .R(1'b0),   // 1-bit reset input
      .S(1'b0)    // 1-bit set input
   );

 

    问题便解决了，有一点需要注意的是ODDR2的输出必须直接连到输出pad上，不可以再引进逻辑内部

 

   在该项目中，我是将专用时钟引脚输入的时钟信号进入IBUFG，出来的信号进如DCM，从DCM出来的信号进入BUFG，再从这里的信号进入ODDR2， 出来后就直接输出去了，但是我试了一下，去掉BUFG，直接将DCM出来的信号送给ODDR2，也能够照常进行，因为ODDR2是将时钟管理IP分出来的 时钟线直连到一些FPGA引脚，而BUFG的信号使用方法在我另一篇博文里面有详细介绍，他们功能我不知道具体有什么不一样的地方，还望大家自己查询下！
