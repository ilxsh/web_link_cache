日期：2018-01-12

但是，重新挂载192.168.2.222这个服务器共享出来的NFS时，依然报错，只是这里有了具体的服务器IP信息。

1
2
3
[root@gzzabbix ~]#  mount  192.168.2.222:/isofile/cdrom64  /mnt/centos64
mount.nfs: access denied by server while mounting 192.168.2.222:/isofile/cdrom64
[root@gzzabbix ~]#

查看NFS共享端的/etc/exports配置文件

1
2
[root@dbrecover ~]# cat /etc/exports
/isofile/cdrom64 192.168.2.*(ro)

在阅读相关资料和多次尝试之后，发现问题为IP格式错误。我想要把192.168.2.22的/isofile/cdrom64共享给同一个网段的所有服务器，正确的写法应该是【192.168.2.0/255.255.255.0】，而不是【192.168.2.*】。

CentOS支持的格式如下：
192.168.100.10、192.168.100.0/24、192.168.2.0/255.255.255.0或者也可以,不支持类似192.168.2.格式配置。

NFS共享端服务器的/etc/exports正确配置后，终于可以正常的mount起nfs。

1
2
[root@dbrecover ~]# cat /etc/exports
/isofile/cdrom64 192.168.2.0/255.255.255.0(ro)
