最近研究了Openvpn好久。现在写一点心得出来。
客户需求：看香港某几个网站使用openvpn翻墙过去，其它所有访问不能用翻墙，要走本地。

系统环境：
Centos x64位 6.8，
使用在线yum安装

一、服务器端配置
详细配置参考其它配置文档
[iyunv@openvpn01 openvpn]# rpm -qa | grep openvpn
openvpn-2.3.11-1.el6.x86_64

生成ta.key文件,用于tls-auth认证。
# openvpn --genkey --secret /etc/openvpn/easy-rsa/keys/ta.key


openvpn配置文件

vi /etc/openvpn/server.conf
port 443
proto udp
dev tun

ca /etc/openvpn/easy-rsa/keys/ca.crt

cert /etc/openvpn/easy-rsa/keys/server.crt
key /etc/openvpn/easy-rsa/keys/server.key
dh /etc/openvpn/easy-rsa/keys/dh2048.pem

server 10.8.0.0 255.255.255.0

push "route 121.40.21.118 255.255.255.255"
push "route 104.27.202.91 255.255.255.255"

tls-auth /etc/openvpn/easy-rsa/keys/ta.key 0 # This file is secret

ifconfig-pool-persist ipp.txt
duplicate-cn
keepalive 10 120

comp-lzo
persist-key
persist-tun

status openvpn-status.log
log    openvpn.log

verb 3
=====================================================
push "route 121.40.21.118 255.255.255.255" 为123cha.com网站查IP的WAN　IP。
push "route 118.184.180.46 255.255.255.255"　为ip.cn网站查IP的WAN　IP。
在客人要求时，改成他需要的HK 网站IP即可，此二行表示路由走VPN访问.

iptables需要做端口转发，openvpn才能使用，请参考其它文档。
最后启动服务
chkconfig openvpn on
service openvpn start

二、客户端软件及配置

在服务器中生成的客户端证书和配置文件，客户端需要的一共有5个文件：ca.crt、client.crt、client.key、ta.key(如果不开启tls-auth，则无需该文件)、client.ovpn。使用Notepad++ 编辑软件，打开上面所说的四个证书与key文件内容，分别粘贴到client.ovpn相对应的地方。
此方法在Linux、Windows、Mac OS下都通用！！

client.ovpn内容及格式如下

client
dev tun
proto udp
remote 114.112.224.61 443
resolv-retry infinite
nobind
persist-key
persist-tun
ns-cert-type server
comp-lzo
verb 3

remote 114.11.224.61  443表示我HK openvpn IP,使用udp 443进行连接。
安卓系统中安装好openvpn-1.1.17-76.apk。将制作好的client.ovpn复制到TF卡中，再启动openvpn导入即可.
依次选择“Import”---"Import Profile from card ",找到client.ovpn，导入。
测试效果
打开浏览器，分别打开三个查WAN IP的网站。123cha.com  ip.cn   ip138.com 前2个网站显示出的是HK openvpn IP. 后面ip138显示出来的你本地运营商的WAN IP。
这样就实现了网站分流，不需要走VPN的，全走本地网络，只有指定的网站才会走VPN。


以上，openvpn client 是使用的证书+key方式认证拨通VPN的。
测试中发现openvpn，在安桌系统在使用mysql帐号、密码认证方式时，并不能完成“指定的网站才走VPN”.但在windows系统里面又是正常的。
下面是服务器端server.conf使用Mysql 帐号、密码时，加的四行参数
plugin /lib64/security/openvpn-auth-pam.so openvpn_mysql
client-cert-not-required
username-as-common-name
auth-nocache
