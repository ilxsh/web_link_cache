OpenVPN：TLS错误：TLS密钥协商未能在60秒内发生

时间  2018-11-18
标签 openvpn 栏目 VPS
原文   https://serverfault.com/questions/92312/openvpn-tls-error-tls-key-negotiation-failed-to-occur-within-60-seconds
http://openvpn.net/index.php/open-source/documentation/howto.html页面告诉错误是因为：“此错误表示客户端无法与服务器建立网络连接.”但我认为这不是这种情况.

这是我的配置：
OpenVPN Server已安装在具有10个有效IP地址的Windows 2008上.我可以从外面ping OpenVPN IP地址,服务器上也没有安装防火墙来阻止端口.

OpenVPN Server Configuration:
local 96.31.90.174
port 1937
proto tcp
dev tun
ca ca.crt
cert server.crt
key server.key
dh dh1024.pem
ifconfig-pool-persist ipp.txt
keepalive 10 120
comp-lzo
persist-key
persist-tun
status openvpn-status.log
verb 3

OpenVPN客户端配置：
我已将它安装在没有防火墙的Windows 7上.

client
dev tun
proto tcp
remote 96.31.90.174 1937
resolv-retry infinite
nobind
persist-key
persist-tun
ca ca.crt
cert client1.crt
key client1.key
ns-cert-type server
comp-lzo
verb 3

看起来你在客户端和服务器之间获得了良好的TCP连接.为了排除任何PKI问题,我将从static key configuration开始,一旦有效,就转到公钥配置.
