debian 7下编译安装最新openVPN
原

 hiwill 发布于 2016/05/27 15:27
字数 822
阅读 1145
收藏 4
点赞 0
评论 0

为什么80%的码农都做不了架构师？>>>   

鄙人（也就是我）个人理解openVPN是在虚拟网卡的基础上使用SSL/TLS协议进行加密的一种vpn技术。SSL即Secure Socket Layer，安全套接字层，位于HTTP协议层和TCP协议层之间。OpenVPN依赖OpenSSL库，加密用；LZO库，数据压缩用。

实现平台：debian 7 x86_64

apt-get update

apt-get upgrade -y

apt-get install -y libssl-dev liblzo2-dev libpam0g-dev git

wget http://swupdate.openvpn.org/community/releases/openvpn-2.3.11.tar.gz

tar xzvf openvpn-2.3.11

cd openvpn-2.3.11.tar.gz

./configure

make && make install


mkdir /etc/openvpn
cp -r /root/openvpn-2.3.11/sample/ /etc/openvpn/
cp /etc/openvpn/sample/sample-config-files/server.conf /etc/openvpn/ 

git clone git://github.com/OpenVPN/easy-rsa

cp -R easy-rsa/ /etc/openvpn/

cd /etc/openvpn/easy-rsa/easyrsa3/

cp vars.example vars

vi vars  根据实际情况，修改以下字段：

set_var EASYRSA_REQ_COUNTRY     "cn"
set_var EASYRSA_REQ_PROVINCE    "beijing"
set_var EASYRSA_REQ_CITY        "chaoyang"
set_var EASYRSA_REQ_ORG         "none"
set_var EASYRSA_REQ_EMAIL       "hihell@163.com"
set_var EASYRSA_REQ_OU          "My OpenVPN"

初始化
./easyrsa init-pki

创建根证书
./easyrsa build-ca
在此过程中需要输入根名称及根密码，需要记住根密码，在以后创建新用户时需要用到
 
创建服务器端证书
./easyrsa gen-req server nopass 在此需要输入服务端名称,不输就是默认的server

签约服务端证书  
./easyrsa sign server server 在此步骤需要输入根密码 

创建Diffie-Hellman parameters 
./easyrsa gen-dh  

将相关文件拷贝到根目录下  
cp /etc/openvpn/easy-rsa/easyrsa3/pki/ca.crt /etc/openvpn  
cp /etc/openvpn/easy-rsa/easyrsa3/pki/private/server.key /etc/openvpn 
cp /etc/openvpn/easy-rsa/easyrsa3/pki/issued/server.crt /etc/openvpn 
cp /etc/openvpn/easy-rsa/easyrsa3/pki/dh.pem /etc/openvpn

创建客户端证书
./easyrsa gen-req client1(名字可自己定义)
过程中记住自己输入的密码

签约客户端证书
./easyrsa sign client client1

将得到的client1.req导入然后签约证书
./easyrsa import-req ./pki/reqs/client1.req client1(前后导入名字必须一致，输入根密码)


配置服务器端

vi /etc/openvpn/server.conf

local X.X.X.X  # vps IP
port 53  #默认是1194
proto udp
dev tun
ca /etc/openvpn/ca.crt
cert /etc/openvpn/server.crt
key /etc/openvpn/server.key   # This file should be kept secret
dh /etc/openvpn/dh.pem
server 10.8.8.0 255.255.255.0  #隧道ip，不能跟本地ip冲突
ifconfig-pool-persist ipp.txt
push "redirect-gateway def1 bypass-dhcp"
push "dhcp-option DNS 8.8.8.8"
push "dhcp-option DNS 8.8.4.4"   #推送谷歌dns到客户端
keepalive 10 120           #连接测试,意思是10秒ping一次,120秒没有返回就断开重连
comp-lzo               #开启压缩传输
duplicate-cn         #如果你要多人共用一个证书的话，请开启本项
max-clients 100
persist-key
persist-tun
log /var/log/openvpn.log    #log日志存放目录
status /var/log/openvpn-status.log
verb 3

sysctl -w net.ipv4.ip_forward=1
iptables -t nat -A POSTROUTING -s 10.8.0.0/24 -o venet0 -j SNAT --to-source vpsip

重启OpenVPN
service openvpn restart
lsof -i:53   #根据端口查看进程信息

ifconfig tun0，这时你会看到新创建了虚拟网卡tun0接口

Linux客户端配置：

client  
dev tun  
proto udp  
remote yourserverip 53  
resolv-retry infinite  
nobind  
persist-key  
persist-tun  
ca ca.crt  
;cert client.crt  
;key client.key  
tls-auth ta.key 1  
auth-user-pass  
ns-cert-type server  
comp-lzo  
verb 3  


扩展：

radiusplugin是radius的一个插件，可以让openvpn使用radius服务器来验证

apt-get install -y g++ libgcrypt11-dev 
wget http://www.nongnu.org/radiusplugin/radiusplugin_v2.1.tar.gz  
tar -zxvf radiusplugin_v2.1.tar.gz  
cd radiusplugin  
make  
cp radiusplugin.so /etc/openvpn  
cp radiusplugin.cnf /etc/openvpn  

编辑radiusplugin.cnf

vim /etc/openvpn/radiusplugin.cnf  

server  
{  
# The UDP port for radius accounting.  
acctport=1813  
# The UDP port for radius authentication.  
authport=1812  
# The name or ip address of the radius server.  
name=radiusserverip  
# How many times should the plugin send the if there is no response?  
retry=1  
# How long should the plugin wait for a response?  
wait=1  
# The shared secret.  
sharedsecret=testing123  
}

在/etc/openvpn/server.conf里追加以下三行

#指定radiusplugin.so插件
plugin /etc/openvpn/radiusplugin.so /etc/openvpn/radiusplugin.cnf
client-cert-not-required
username-as-common-name
