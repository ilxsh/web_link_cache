bitnami redmine配置全过程

常见问题：

我在自己的机器上面配置完毕以后，移植到另外一台机器上面，登陆页面总是在检查network，并且最后网络加载失败，不论我是用桥接还是NAT方式连接。登陆系统以后，我尝试连接网络失败，尝试执行以下命令重启network

sudo /etc/init.d/networking restart

但是执行完毕以后报错“connot find device eth0。”

以前我碰到这样的问题，只需要修改配置文件
sudo vim /etc/network/interfaces

将其中的eth0修改为eth1，然后再重启网路就好了，但是这回不行。

最后找到解决方法如下：
http://www.serenux.com/2009/11/howto-fix-a-missing-eth0-adapter-after-moving-ubuntu-server-from-one-box-to-another/
打开文件
sudo nano /etc/udev/rules.d/70-persistent-net.rules
删除以下记录：
# PCI device 0x8086:0x1004 (e1000)

 

 基础配置：

参考官方的quick start手册。
http://wiki.bitnami.com/Virtual_Appliances_Quick_Start_Guide#How_to_install_Webmin.3f
需要注意的：
1.默认自带的vsftpd只绑定了127.0.0.1:21端口，需要修改sudo vim /etc/vsftpd.conf当中的内容，将其监听地址修改。
2.里面的源是国外的，网速特别慢，建议修改为国内的源，我使用的就是163的源，具体源地址搜索ubuntu source list。

测试是否部署完毕：

/opt/bitnami_redmine/ctlscript.sh status
访问地址：http://192.168.52.240:80/。如果没有修改apace的端口的话，默认是80端口。
