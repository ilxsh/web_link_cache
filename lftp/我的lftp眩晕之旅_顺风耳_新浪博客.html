我的lftp眩晕之旅

(2013-03-30 17:37:50)
标签：

it

 分类： SAP_Basis
我的lftp眩晕之旅


    我在Win7中(VM Client)下载了SAP ECC6.0 IDES For Linux+Oracle的安装光盘，解压后，如何复制到ESXi VM中呢？网友介绍我用lftp，我在win7中使用了SecureCRT成功连到VM(RHEL 6.3)，然后运动lftp命令，后来想想这样很怪，其实在win7中开好FTP服务，直接在VM(RHEL)下使用lftp命令来从win7中取这些安装介质不就完了？
   以下是我为了现实安装介质从client传送到esxi下vm的过程：


(1)直接使用
[root@rels63 ides]# lftp
-bash: lftp: command not found

(2)安装lftp
yum install lftp -y
后，可以使用lftp了，但输入合作命令均提示:
[Delaying before reconnetc: 1]

安装完毕，检查：
$ rpm -qa|grep vsftpd
vsftpd-2.2.2-11.el6_3.1.x86_64

再运行lftp -u jason,abc123 192.168.0.111
提示：Delaying before reconnect


(3)网上说要修改配置文件
vi /etc/lftp.conf
在其最末尾添加下列三行后保存退出
set ftp:passive-mode off
set file:charset utf8
set pget:default-n 5

set ftp:passive-mode no #关闭被动模式转为主动模式
编码设置（避免使用中文时出现乱码，其实可能也不需要配置这里，因为我安装SAP IDES其间并不使用中文）：
set ftp:charset "gbk"
set file:charset "UTF-8"

再运行lftp -u jason,abc123 192.168.0.111
仍然提示：Delaying before reconnect


(4)怀疑是防火墙在捣鬼，于是关闭防火墙
先检查在哪些运行级别是on状态的：
chkconfig --list|grep iptables

尝试停止：
service iptables stop
如果想让防火墙不随机启动
chkconfig --level 2345 iptables off

删除防火墙规则文件
rm /etc/sysconfig/iptables

再运行lftp -u jason,abc123 192.168.0.111
仍然提示：Delaying before reconnect

(5)会不会是默认的21端口未开呢？
在win7下cmd  telnet 192.168.0.111，结果是不通的。
通常，FTP默认端口为：21，20 ，telnet是23端口，SSH服务默认使用的端口是22。
于是，想到打通这个端口，以下是安装telnet服务的过程，事后，想想，这一步可能是不需要的，

检查有无安装telnet，
rpm -qa telnet
rpm -qa telnet-server

没有就用yum安装
yum install telnet -y
yum install telnet-server -y

配置telnet服务配置文件
vi /etc/xinetd.d/telnet
在disable=yes前加一个#，将其注释掉即可。
安装telnet-server后，系统才有文件/usr/sbin/in.telnetd

由于telnet服务是由xinetd守护的，需要重启xinetd守护进程，所以安装完telnet-server，要启动telnet服务就必须重启xinetd
#service xinetd restart
或
#/etc/init.d/xinetd restart
注意：能够自己重启的服务都不需要xinetd，像ftp，samba这些服务不需要xinetd，像telnet才需要xinetd。

再回到win7中cmd telnet 192.168.0.111就通了！

如果想让telnet默认开机启动
可以用ntsysv激活telnet
或
chkconfig --add telnet
chkconfig telnet on

要注意的是telnet登陆时是明文传送密码，不安全，建议使用ssh(secure shell)替代，Win下一般用F-Secure SSH或putty SSH。
附注：
netstat –apn #查看所有端口占用情况
netstat –apn | grep 21 #看21端口有无使用
ps -aux | grep 21 #看21端口被什么应用再用，有时可以找到PID后，Kill了它

再运行lftp -u jason,abc123 192.168.0.111
仍然提示：Delaying before reconnect

(6)会不会是RHEL 6.3的ftp服务跟本没开？
注：后来想想，觉得挺可笑，FTP-Server是Win7才对，FTP-Client应是RHEL
安装并启动FTP服务
yum install vsftpd -y
其配置文件：/etc/rc.d/init.d/vsftpd
启动此服务:
#service vsftpd start

再运行lftp -u jason,abc123 192.168.0.111
问题解决了，成功登陆并可以使用各个命令了。

(7)尝试上传Win7中的安装介质到VM中。
运行下面的命令，可是提示错误：
ftp jason@192.168.0.111:/mnt/ides> mirror -R E:\unzip\51041939_1-6_RDBMS /mnt/ides
mirror: Access failed: /home/jason/E:\unzip\51041939_1-6_RDBMS: No such file or directory
1 error detected

(8)FTP-Client与FTP-Server搞错对象
我将(7)的错误发给网友，帮我看下，它发来了它正常操作的过程：
[root@svr~]#lftp
lftp :~> open 192.68.0.10
lftp 192.68.0.10:~> login administrator
Password:
lftp administrator@192.68.0.10:/> ls -l
..............................
.....显示出目录及文件夹.....
..............................
lftp administrator@192.68.0.10:/> mirro 51041987
Total: 1 directory, 121 files, 0 sylinks

突然醒悟，我将FTP-Client与FTP-Serve搞晕了。
Win7作为Client，它有SAP安装介质，就是FTP-Server，而VM是FTP-Cllient，我应在EHLX中发起对Win7的FTP访问并“下载”安装介质才对！
我原来的想法是：在Win7中运行SecureCRT，通过SSH登陆到VM的RHEL，然后使用lftp命令，想将Win7中的安装介质“上传”到VM中。

(8)掉转枪口
[root@rels63 ides]# lftp
lftp :~>open 192.168.0.100
lftp 192.168.0.100:~>login administrator
Password:
lftp administrator@192.168.0.100:~>ls -l
'ls -l' at 0 [Connecting...]
并不通！！！
但[root@rels63 ides]#ping 192.168.0.100是通的！ win7作为FTP-Server，是不是没有开FTP服务呢？

(9)为Win7安装FTP服务
需求背景：程序开发需要使用FTP功能，本地需搭建FTP服务器测试。 
以下过程为简单配置可用FTP测试步骤：
1.控制面板->程序和功能->打开或关闭windows功能,在列表里面找到 internet信息服务,选中"ftp服务",确定(我的经验是三个项全勾选，特别是win7旗舰版).
2.服务安装好后，下面需要设置FTP站点：我的电脑右鍵-管理->Internet信息服务(IIS)管理->在连接树下面添加FTP 站点
3.填写名称、虚拟目录、绑定IP(本地测试用192.168.0.100)、SSL可以选择无,授权允许访问指定本机管理员用户administrator
(因为win7对文件目录访问安全有默认限制，使用administrator用户拥有最高权限)、权限选中“读取”“写入”。
  
以上配置好后，在浏览器输入ftp://192.168.0.100 弹出登陆输入用户和密码框即表示FTP连接设置成功。
容易遇到的问题：指定用户没有选择administrator,其它用户没有对设置 的虚拟目录有足够的访问权限 ，以致ftp访问不通。
"系统和安全"不好找，方法是：控制面板的选项 “类别”“大图标”“小图标”改为 类别第一个就是。

经过上面的步骤，Win7下ftp服务建好了，可是仍然从vm中不能lftp成功过来！投降了！


(10)直指佛心
我们要做一件事，有很多方法，必须集中注意力，精力去实现既定的目标！我的目的是安装SAP IDES EHP6，而不是研究lftp，
我想到用CuteFTP将win7中的安装介质直接upload到vm中就算了，这样一试，果然很快！






【附录】：
(1)lftp上传/下载目录
lptp成功登陆ftp server后：
下载：mirror rdir ldir　　// 将远程目录rdir下载到本地目录ldir
上传：mirror -R ldir rdir　　// 将本地目录ldir上传到远程目录rdir

(2)如何远程登录Linux
远程登陆linux你可以采用：
一、用ssh登陆。可以采用SecureCRT、SSH Secure Shell Client、WinSCP等软件，前提是你的linux要安装和启用ssh，大部分的linux应该安装完后就可以使用了。
二、用telnet登陆。方法与上面类似，linux开启telnet。
三、远程桌面。linux安装VNC服务端，作相应配置，推荐你用TightVNC，你可以在windows下用TightVNC Viewer客户端登陆，也可以在安装有JAVA后的浏览器中登陆，进行远程管理。


     反思，千万不要掉到技术的陷阱里去，其实很多东西，我们只需要大概知道其原理，然后简单去测试，就知道哪里有问题了，再就是，做技术活儿，要有强烈清晰的目的性，我们的目标是什么？不管什么方法，直接先实现目标再说。
