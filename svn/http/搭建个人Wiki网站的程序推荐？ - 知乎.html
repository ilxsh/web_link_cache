欢迎使用 amWiki轻文库


作为 Atom 编辑器的一款插件，所有编辑工作在 Atom 编辑器中完成，一键生成 html5/js 自适应网页，所有文档使用 Markdown 标记语法并存储为“.md”扩展名文本文件，只需 http 纯静态网页空间，无需服务端动态语言环境也不使用数据库。

----------
16年前初的时候，也是在知乎这个问题下，我发现了前端纯静态项目 mdwiki，了解了一番感觉由于页面布局问题文档篇数容量有限。
页面布局其实好解决，网络上的文档随处可见，当时印象较深的是微信公众平台开发者文档，仿一下就是了，于是萌发了自己重新写一个的想法。
但是转身又一想，由于只是一些前端 html/js 的页面，其实实用性还是有局限性的，毕竟如果每添加一篇文档都要手工改导航，写文档时图片都要自己整理文件与路径等等，岂不是很麻烦？
作为一枚 FEer，当然想到用了 node，但是用 node 做服务器开发很快就否定了，因为不够“轻”。
然后又继续想到，既然要用编辑器来编辑文档，为何不在编辑器中就搞定这一块？在 node 与“编辑器”两个关键词重叠的条件下，自然而然的就找到了 Github 的开源文本编辑器 Atom。
于是在这些考量下，造了 amWiki 这个新轮子。

强烈推荐mediawiki，功能强大到无以复加，也有很多的插件。

也要说说缺点，国内玩这个的不少，但是缺少深度的研究，特意针对中文圈的插件少一些，上手还不算太难，但是深入研究可能需要一些技术功底。

我在我的mediawiki站点中写了我的一些感受：

1.随时记录和整理，不断修改和完善的编辑思维。

使用mediawiki的时候，有了想法就可以开始录入和编辑了，不需要思索很多，在wiki中，强调一种不断完善的思维，不需要编辑完善以后在发布，wiki是一个不断完善的过程。这样有利于我们有了想法之后立即着手编辑，这样有利于我们生产更多的内容。其实，一篇有价值的词条，不就是在不断编辑完善的过程中产生的吗？

2.这是一个知识系统

方便快捷的建立内部链接，通过分类不但建立起系统性更强的知识体系，很适合用来建立起自己的知识体系。mediawiki的发散性很强，它不像blog系统，它不是线性的发展模式，而是以点到线，以线到面的发散性扩展模式，扩展性很强，就如同一个思维导图一般，能够技法头脑的风暴。

3.快速建立内部联系

利用mediawiki的标记语言，可以快速的建立内部链接、分类，从而可以方便的加强内部各个页面之间的联系，这是有利于SEO的，也是有利于知识网络系统的建立，有利于将分散的页面整合为一个更加紧密的系统，这一点是其他CMS系统都难以比拟的。

4.良好的协同机制

mediawikidi支持多用户，有较好的权限管理机制，有利于参与者在一个框架之中方便发挥自己的专长，为系统的内容建设做出自己的贡献。编辑、讨论，更多的参与形式，mediawiki不仅是一个内容组织和生产工具，也是一个简易的内容生产交流社区。

5.良好的安全保障机制

mediawiki有很好的历史版本功能，对于一些不好的编辑版本，可以方便的撤回，减少误操作所造成的损失。用户的权限机制也不错，可以进行权限的设置，保证社区的安全秩序。

6.强悍的性能

mediawiki采用数据库的存储方式，将数据和页面分离，利用php和mysql的强大技术支持，性能更强。现在是世界上使用最广泛的wiki系统，维基百科利用它建立起了世界上最为丰富的百科系统，在很多大型的商业网络中，mediawiki也能很好的胜任安排给它的任务。

7.更高的效率

mediawiki的“模版”功能可以插入动态变化的内容，让那些重复性的编辑工作变得一劳永逸——一处修改后，其他位置插入的相应的模版也会一同发生变化。
