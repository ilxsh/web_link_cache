[文章]Windows server 2016之基于虚拟机的桌面部署

已采纳 收藏
 0  2309 0

【摘要】

本文主要简单介绍Windows Server的远程桌面服务（RDS），并在实验环境中做简单的测试。

【正文】

一   基本概述

远程桌面服务（RDS）是为每个最终用户需求构建虚拟化解决方案的首选平台，包括提供单独的虚拟化应用程序，提供安全的移动和远程桌面访问以及为最终用户提供从云运行其应用程序和桌面的能力。

RDS提供了部署灵活性，高成本效益和可扩展性。

RDS解决方案包括基于会话的虚拟化、虚拟桌面基础架构（VDI）。可以单独使用两种方案，或者将它们结合使用：

l  基于会话的虚拟化：利用Windows Server的计算能力提供经济高效的多会话环境，以承载用户的日常工作负载

l  VDI：利用Windows客户端提供用户期望的Windows桌面体验所带来的高性能，应用程序兼容性和熟悉程度。

在这些虚拟化环境中，面向用户发布的内容具有更多的灵活性：

桌面：通过安装和管理的各种应用程序，为用户提供完整的桌面体验。

RemoteApps：指定在虚拟机上托管或运行的个别应用程序，但看起来像在本地应用程序中一样运行在用户的桌面上。这些应用程序有自己的任务栏条目，可以调整大小，并跨显示器移动。这适合在安全的远程环境中部署和管理关键应用程序，同时允许用户使用和定制自己的桌面。

详情可查看官网：https://docs.microsoft.com/en-us/windows-server/remote/remote-desktop-services/welcome-to-rds

 

二   部署RDS基础架构

本次实验拓扑如下：


RDS运行在Windows的域环境中（本文档中为lab.com域），部署前请确保完成服务器重命名，配置静态IP，加域等操作，以保证部署过程顺利完成

2.2       添加服务器

1)     登录RDCB1.lab.com，打开服务器管理器界面，点击管理——添加服务器；


2)     在添加服务器窗口中，搜索添加RDVH1、RDWA1和RDSH1，点击确定；


3)     添加完成后可以在服务器管理器的所有服务器界面看到添加的服务器；


2.3       添加角色

1)     打开服务器管理器，在点击管理——添加角色和功能；


2)     点击下一步；


3)     在安装类型选项中，选择远程桌面服务器安装，点击下一步；


4)     在部署类型选项中，选择标准部署，点击下一步；


5)     在部署方案选项中，可以根据要部署的环境选择部署方案，此处选择基于虚拟机的桌面部署，点击下一步；


6)     此页面说明此次部署将安装的角色服务，确认当前使用的用户是否具有部署权限，点击下一步；


7)     指定RD连接代理服务器；此处选择RDCB1.lab.com为连接代理服务器，点击下一步；


8)     指定RD Web访问服务器；此处选择RDWA1.lab.com为RD Web访问服务器，点击下一步；


9)     指定RD虚拟化主机服务器；此处选择RDVH1.lab.com为RD虚拟化主机服务器，点击下一步；RDVH1服务器需要提前安装Hyper-V服务。


10)    确认安装信息无误后，勾选需要时重新启动目标服务器，点击部署；


11)    等待部署完成。


 

2.4       创建虚拟桌面集合

2.4.1       准备模板机

在创建虚拟机桌面集合之前，需要准备一个模板机作为创建虚拟机的模板，这台机器应该是打好最新补丁，各种配置，并且做了sysprep的。此处不一一描述，请按照自身需要制作模板即可。

2.4.2     创建虚拟桌面集合

1)     打开服务器管理器——远程桌面服务——集合，选择任务，点击创建虚拟桌面集合，如下图所示。


2)     点击下一步。


3)     为集合命名，点击下一步；


4)     指定集合类型，此处实验选择虚拟机集合池，点击下一步；


5)     选择提前准备虚拟机模板，模板机需要处于关机状态，点击下一步；

6)     PS：在Windows Server 2012中，模板机必须是第一代虚拟机，在Windows Server 2016则可以使用第二代虚拟机作为模板。


7)     指定虚拟机设置；此实验选择提供无人参与安装设置；点击下一步；


8)     为即将新建的虚拟机选择合适的时区，以及OU，点击下一步；


9)     指定具有访问权限的用户组，以及虚拟机名称的前缀和后缀，和集合中的虚拟机数量。点击下一步；


10)   指定虚拟化主机，以及每个虚拟化主机上要创建的虚拟机数量。点击下一步；


11)   指定虚拟机存储的路径，点击下一步；


12)   选择是否启用用户配置文件磁盘。如果启用，需要指定一个共享路径来存放用户配置文件，并且需要对该文件夹有足够的权限。点击下一步；


13)   确定集合的配置信息。点击创建；


14)   等待虚拟桌面集合创建完成。


15)   集合创建完成。


16)   可以在RDVH1虚拟主机服务器上看到创建的虚拟机。


2.4.3       测试连接

17)   打开浏览器，输入http://rdwa1.lab.com/rdweb，rdwa1.lab.com位为RD Web访问服务器，输入账号密码。如下图所示。


18)   在页面中可以看到该用户具有权限的资源，此处点击集合VMs，如图所示；


19)   在弹出窗口中选择设置选项，点击连接；


20)   输入账号密码，并等待连接完成。

  

21)   可以看到已连接到VMs集合分配的虚拟机VMs-1；
