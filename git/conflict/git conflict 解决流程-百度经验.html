git conflict 解决流程

关于git confilict，我自己有常用的解决方法，在这分享给大家。

方法/步骤

  • 出现git conflict一般是服务器上有A的新提交，而此时B也进行了修改，如果这时使用git pull的时候，B的修改有跟A的修改到同一文件，那么就有冲突出来。

  • 出现这种情况的git conflict该如何处理呢？

  • 首先，将提示冲突的文件复制出来备份着；

  • 接下来使用git checkout patch_to_confilict_file将冲突文件checkout掉；

  • 接下来再使用git pull，如果此时还有冲突，那么重复上面的步骤直至没有confilict出现；

  • 当冲突解决后，我们再将备份的文件中的修改项合并到相应的文件中；

  • 此时再将修改的文件使用git add添加到本地库；

  • 接下来再使用git commit进行提交；

  • 再使用git push将本地提交上传到服务器。

  • 至此就完成了一次confilict的处理。
